**Design and Implementation of Parking Lot Management System Based on License Late Recognition**

## 摘 要

随着当代人民生活水平的不断提高,越来越多的家庭会去购买汽车用来驾驶, 作为平时的出门方式。然而随着我国汽车数量的不断增长,驾驶汽车出门的人们 经常会遇到最头疼的问题——停车难。为了解决这个问题,全国各地政府开始对 停车位数量重视起来,新建很多公共停车场。然而传统的停车场依然存在着很多 的问题,例如:停车位数量不够,进出口拥堵,效率低下,需要雇佣专门的的管 理人员在停车场的出入口进行指挥、引导,给运营单位带来很多的人工成本和管 理难题。由于传统停车场拥有以上的种种问题,不足以满足现代的停车场需求, 我设计了一个基于车牌号识别的停车场管理系统。

该系统使用开源的 Paddle-OCR 作为实现车牌号识别的核心技术,使用 Paddle-OCR 工具库中已经训练好的模型去识别车牌号。系统采用主流的前后端 分离技术,前端使用 vue,并借助开源的 element-admin 后台管理界面模板实现 系统的前端页面开发。后端使用 java 语言,并采用国内最流行的微服务框架 SpringCloud-Alibaba 进行后端开发。同时使用 RabbitMQ 消息队列用来进行异步 处理。数据库采用关系型数据库 MySQL。服务部署使用的是腾讯云 Debian 11 云 服务器进行服务的部署。

关键词:停车场管理系统 SpringCloud 车牌识别

## **Abstract**

With the continuous improvement of contemporary people's living standards, more and more families will buy cars to drive as their usual way of going out. However, with the continuous growth of the number of cars in our country, people who drive cars often encounter the most headache-the difficulty of parking. In order to solve this problem, governments all over the country began to pay attention to the number of parking spaces and built many new public parking lots. However, there are still many problems in the traditional parking lot, such as insufficient number of parking spaces, congestion in entrances and exits, and inefficiency, so it is necessary to hire special managers to direct and guide the entrances and exits of the parking lot. It brings a lot of labor costs and management problems to the operating units. As the traditional parking lot has the above problems, which is not enough to meet the needs of modern parking, I designed a parking lot management system based on license plate recognition.

The system uses the open source Paddle-OCR as the core technology of license plate recognition, and uses the trained model in the Paddle-OCR tool library to recognize the license plate. The system uses the mainstream front-end separation technology, the front-end uses vue, and with the help of open source elementadmin background management interface template to achieve the front-end page development of the system. The back-end uses Java language, and uses SpringCloud-Alibaba, the most popular micro-service framework in China, for backend development. RabbitMQ message queues are also used for asynchronous processing. The database adopts relational database MySQL. Tencent Cloud Debian 11 CVM is used for service deployment.

**Key words :** Parking lot management system SpringCloud license plate recognition

## 绪 论

#### (1) 研究背景以及意义

随着人们生活水平的不断提高,越来越多的家庭选择汽车作为日常出行的代 步工具,汽车几乎成为了现代人们的生活必需品。

近年来,我国汽车保有量呈现出持续快速增长的势态,根据公安部 2022 年 2 月 11 日发布 2021 年全国机动车和驾驶人数据,2021 年全国机动车保有量达 3.95 亿辆,其中汽车 3.02 亿辆﹔机动车驾驶人达 4.81 亿人,其中汽车驾驶人 4.44 亿人。新能源汽车保有量达 784 万辆,同比增长 59.25%。社会车辆数量正在高 速增长。

#### (2) 国内外研究现状

国内停车场管理系统是伴随着国内公用停车场的大量出现而产生并逐步发 展起来的。最初的国内停车场管理系统是在引进和消化吸收国外同类系统的基础 上逐步发展起来的,并在此基础上不断改进提高。目前,国内主要采取的技术有: 读卡技术、卡票结合技术、车牌识别技术。

许多国外停车场管理系 统还配备了停车车位引导系统、停车车位查询系统 等智能化设备,使停车场管理系统的功能更加完善和丰富。

#### (3) 研究内容

通过车牌识别技术实现停车场的智能化管理,减少运营单位的管理成本,提 高泊车效率,减少拥堵情况。

## 1 需求分析

## <span id="page-8-2"></span><span id="page-8-1"></span><span id="page-8-0"></span>1.1 功能需求

#### 1.1.1 普通用户需求

普通用户指的是使用停车场进行停车的用户,他们将使用通过注册功能注册 账号;使用账号登录系统;通过自身车辆的车牌号进行车辆的入场操作。用户可 以选择空余的停车位进行停车,在停车结束后通过车辆车牌号进行车辆的出场, 并可以进行停车费的支付。

具体需求如下:

注册:通过注册功能填写用户信息进行用户的注册。

登录:输入登录账号以及密码,登录系统。

车辆进场:查看不同停车场的分类,选择新能源充电停车位或普通停车位, 通过识别车牌号记录车辆的车牌号,选择停车场进行入场。

车位选择:在入场后查看所有空闲车位,选择一个车位进行车辆的停放。

车辆出场:通过识别自身车辆的车牌号出停车场,系统需发送邮件给用户通 知用户缴费金额以及缴费方式。

缴费:通过系统进行支付缴费,缴费成功后系统需发送邮件通知用户缴费成 功。

信息更改:用户可以随时更改自己的名称、邮箱、手机号等信息。

#### <span id="page-8-3"></span>1.1.2 管理员用户需求

管理员用户指的是使用停车场管理系统的管理员,他们具有很高的权限,在 登录系统后可以查看停车场的当日运营情况,停车场管理、停车位管理、用户管 理,导出当月订单的 Excel 表格。

具体需求如下:

登录:输入登录账号以及密码,登录系统。

查看当日运营情况:在登录系统后,查看当日的运营情况,例如:当日订单 量、当日收益、当日停车场停车位使用情况。

停车场管理:可以查看所有的停车场信息,并对它们设置每小时停车费;可 以新增停车场,也可以删除停车场。

停车位管理:可以查看所有的停车位信息;可以新增停车位,设置其所属于 的停车场;可以删除停车位。

用户管理:管理员有权查看所有的用户信息,对用户进行新增,信息的修改, 以及删除操作

月度报表导出:导出当月的订单 Excel 报表,汇总当月总收入。

## 2 系统分析

## <span id="page-10-1"></span><span id="page-10-0"></span>2.1 可行性分析

此系统主要通过图像识别技术实现车牌识别和主流前后端框架进行系统开 发实现车牌识别的停车场管理系统

#### (1) 技术可行性

前端采用 Vue 进行开发,Vue 是一个开源的前端 JavaScript 框架,同时拥有 element UI、Ant Design 等各种第三方前端组件库进行,同时也有 Element-Admin 这种开源的后台管理模板可以使用。通过这些开源项目,可以轻松的进行前台页 面的开发。

后端使用 Java 语言,采用主流的 SpringBoot 框架和 SpringCloud-Alibaba 微 服务生态,使用微服务架构进行开发,相对于传统的单体应用架构,解耦程度更 高,对业务进行了服务拆分,当未来需要更新迭代的时候,只需修改业务所在的 服务包内的代码进行更新替换,不会使整个后端服务宕机。

本系统使用的数据库为 MySQL。MySQL 作为一种关系型数据库,在 Web 应用中足以满足系统需要。同时对比其它的数据库管理系统,MySQL 是开源的。

系统中采用 RabbitMQ 作为消息队列,用来进行异步处理,减少用户等待时 间。

在服务部署上,采用了近些年最流行的云服务器进行部署,相较于传统的实 体机,云服务器可以更好的实现防火墙配置,同时可靠性更高。云服务器获取公 网 IP 地址也相对容易,配合购买的域名绑定使用,访问服务十分便捷。因此本 系统将采用腾讯云的 Debian 服务器进行服务的部署,相较于 Windows Server, Linux 的命令行十分强大,可以做到完全不需要可视化界面,对于一个低内存的 学生版云服务器,更加节省内存空间,让我们在部署前端服务,后端微服务,中 间件等时不用过于担心内存容量的占用。

综上所述,本系统的技术可行性非常之高,对于个人独立开发部署来说十分 合适。

#### (2) 操作可行性

本系统功能十分清晰,作为一个 Web 应用,用户的所有操作都可以在浏览 器中进行,不要用下载单独的客户端进行操作,同时由于使用了 element-admin 作为后台管理界面的模板,系统 UI 十分耐看,用户的使用体验极佳。

因此本系统操作可行性非常之高。

## (3) 经济可行性

本系统中采用的技术几乎全是开源的,使用无需付费,同时在开发中,IDE 的选择也十分重要,后端 IDE 使用了 JetBrains 公司的 IntelliJ IDEA,这是一款付 费 IDE,但是可以使用学信网截图进行认证,认证后即可免费使用。前端文本编 辑器为 VSCode,作为微软下的一款开源文本编辑器,具有很多的开源第三方插 件可以使用,方便我们进行开发。

本系统在开发部署过程中使用所需资金极小,经济可行性十分强。

综上所述,本系统各方面的可行性都十分之高,完全可以进行系统开发。

## <span id="page-13-0"></span>2.4 相关技术

#### <span id="page-13-1"></span>2.4.1 SpringCloud-Alibaba

SpringCloud-Alibaba 提供分布式应用开发的一站式解决方案。它包含开发 分布式应用程序所需的所有组件,包括服务注册与发现、流量控制和服务降级、 分布式配置等各种微服务开发所需要的组件。SpringCloud-Alibaba 中的每个服务 都必须使用 SpringBoot 进行开发。

#### <span id="page-13-2"></span>2.4.2 SpringBoot

SpringBoot 是一个基于 Java 的开源框架,用于创建微服务。相对于传统的 SSM 框架(Spring SpringMVC Mybatis)减少了所需要的配置,SpringBoot 强调 "约定大于配置",采用官方约定的方式编写代码,几乎可以做到完全不用 xml 文件进行 Bean 的配置。

#### <span id="page-13-3"></span>2.4.3 SpringMVC

Spring MVC 是一个 Java 框架,用于构建 Web 应用程序。它遵循模型-视 图-控制器设计模式。它实现了核心 Spring 框架的所有基本功能,例如控制反转、 依赖注入。

#### <span id="page-13-4"></span>2.4.4 Mybatis

MyBatis 是持久层框架,支持自定义 SQL 和动态 SQL 等功能吗,结合 SpringBoot 的自动配置,MyBatis 在使用中只需要在 SpringBoot 的配置文件中配 置好数据源和mybatis的mapper地址,就可以通过xml或者注解的方式编写SQL, 同时支持将 SQL 返回结果映射为 Java 的实体类。

#### <span id="page-13-5"></span>2.4.5 RabbitMQ

RabbitMQ 是一款消息中间件,可以在应用层中传递消息,比如普通文本或 者 JSON 字符串。在本系统中,RabbitMQ 主要用作异步通讯。

#### <span id="page-14-0"></span>2.4.6 Paddle-OCR

Paddle-OCR 是百度飞桨发布的开源文字识别模型套件,是一款十分实用的 文本识别模型/工具库。本系统使用 PaddleOCR 作为实现车牌识别的核心技术。

#### <span id="page-14-1"></span>2.4.7 Docker

Docker 是一个用于开发、发布和运行应用程序的一种工具。将应用与依赖 环境统一打包成一个 Docker 镜像并发布容器,做到开箱即用。

#### <span id="page-14-2"></span>2.4.8 Vue

Vue 是一套用于构建用户界面的渐进式的 JavaScript 框架。与其它大型框架 不同的是,Vue 被设计为可以自底向上逐层应用。Vue 的核心库只注重视图层, 遵循 MVVM 架构 。同时 Vue 拥有各种第三方组件可以使用,极大减少了前端 的开发难度。

#### <span id="page-14-3"></span>2.4.9 MySQL

MySQL 是一款关系型数据库。将数据存储于数据库中不同的表中,是一款 十分优秀的关系型数据库。

#### <span id="page-14-4"></span>2.4.10 nginx

nginx 是一个高性能的 HTTP 和反向代理 web 服务器,本系统中将使用 nginx 进行前端服务部署并进行 http 请求转发处理前后端分离中的跨域问题。

## <span id="page-15-1"></span><span id="page-15-0"></span>3.1 系统总体架构设计

本系统前端采用 Vue 框架。后端使用 SpringCloud-Alibaba 开发的微服务架 构,总体分为 3 个服务,分别为:用户服务,停车场服务,邮件服务;并使用 nacos 进行服务的注册与发现。使用 MySQL 作为数据库进行数据的存储,使用 RabbitMQ 进行异步通信,使用自身搭建部署的 PaddleOCR 车牌识别服务识别车 牌号。系统总体架构图如图 3-1 所示。

#### <span id="page-15-2"></span>3.2 系统功能模块设计

系统功能模块设计是将总体的功能需求划分为不同的功能模块,每个功能模 块组合起来,完成一个系统的完整功能。

本系统主要的功能模块有:用户管理、停车场管理、停车位管理、订单管理、 车辆管理。

用户模块包括:注册、登录、用户查询、信息修改。用户可以点击注册按钮 填写表单信息,点击提交后注册账户。在注册账户之后可以通过输入账号名和密 码后登录系统。若以管理员身份登录系统后,可以通过用户查询功能查看所有用 户的信息,并拥有权限对所有用户的信息进行更改。

停车场模块包括:新增停车场、设置停车场金额、查询停车场。在以管理员 权限登录系统后,可以通过表单填写新增一个停车场,并对停车场进行每小时收 费金额设置。普通用户和管理员用户均可使用停车场查询功能查询所有停车场的 信息(停车场类型、收费金额)。

停车位模块包括:新增停车位、修改停车位信息、查询停车位。在以管理员 权限登录系统后,可以通过表单填写新增一个停车位,并设置其所属的停车场。 同时管理员可对所有的停车位信息进行修改。普通用户和管理员用户均可使用停 车位查询功能查询所有停车位的信息。

订单模块包括:查看订单、发送订单消息、创建订单、导出月度订单、支付 费用。所有用户均可查看订单信息,管理员可查看所有订单的信息,普通用仅可 查看自身订单。在车辆入场后,系统将自动创建一条订单记录,并在成功支付订 单后发送邮件消息通知用户支付结果。管理员可通过导出月度订单功能导出当月 的订单记录,导出格式为 Excel 格式。

车辆管理模块包括:设置车辆信息、车辆进场、车辆出场。用户通过输入车 牌号添加自身车辆;通过上传车牌号图片实现车辆的进场和出场功能。

## <span id="page-21-1"></span><span id="page-21-0"></span>4.1 用户管理模块

用户管理模块包括:注册、登录、用户查询、信息修改这些功能模块。

#### <span id="page-21-2"></span>4.1.1 注册

用户在前端填写表单信息,包括:账号名、密码、手机号、邮箱号进行注册, 点击注册后前端将发送以 JSON 格式为参数的 post 请求到后端 API 服务,后端 收到请求后通过 openfeign 调用用户服务的注册功能。

注册功能被调用后会对密码加盐后使用 MD5 算法进行哈希运算,盐值为配 置文件中定义的盐值;生成 uuid、创建时间、设置默认权限为普通用户。最后在 数据库的用户表中插入一条新的数据,并对插入操作进行 Exception 异常捕获, 捕获到异常时调用自定义的 ResultUtil 工具类的 error 方法生成统一的返回格式, 并将异常信息添加到返回值中,最终将是否成功注册的结果通过 JSON 格式的文 本返回给前端。

## 结论

<span id="page-41-0"></span>本系统成功完成,前端使用 Vue,后端使用 SpringCloudAlibaba 微服务生态, 采用了 MySQL、RabbitMQ,PaddleOCR 中间件。使用了 Docker 搭建的 RabbitMQ 以及 PaddleOCR 服务,并在最后在腾讯云的 Debian Linux 云服务器器上成功部 署运行。

前端借助 Vue 和 ElementAdmin 这种开源后台管理模板,极大程度的降低了 前端的学习成本和开发成本。使得前端的开发过程中顺利了很多。

由于 Netflix 的微服务组件大部分都已不再更新,所以本系统后端采用了阿 里巴巴的微服务解决方案,同时使用 SpringBoot 作为服务开发框架,减少了很多 xml 的配置工作。

通过 Docker 部署的 PaddleOCR 服务极大程度的解决了车牌识别的问题,使 得本系统完全不依靠网络请求第三方 API 调用,加上使用了云服务器,本系统的 稳定性十分之高。

本系统在存储密码使用了 MD5 算法进行哈希计算,MD5 作为一款常用的算 法,导致网络上很多系统的数据库中存储的 MD5 值越来越多,很多简单的密码 的 MD5 运算都已经记录在了其它数据库中,所以本系统对密码进行了加盐操作, 使得出现重复密码的概率更低。

对于数据库的使用,由于本系统的数据量并不庞当,SQL 查询效率相当之 高,所以并未采用 Redis 这种高速的 NOSQL 内存服务器,也未在 MySQL 中为 字段添加除主键索引外的其它索引。

本系统相较于商业使用的系统仍有很多不足之处,功能并不完美。但作为一 个毕业设计的项目来说,实现本系统的过程使得我学会了很多新技术的使用,并 提高了我的代码规范,让我的编码能力有了很大的进步。
