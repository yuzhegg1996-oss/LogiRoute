# 摘 要

近些年来,随着科技的飞速发展,互联网的普及逐渐延伸到各行各业中,给人 们生活带来了十分的便利,疫情常态化下无接触物流配送服务系统利用计算机网络 实现信息化管理,使整个疫情常态化下无接触物流配送服务管理的发展和服务水平 有显著提升。

本系统主要实现无接触物流配送服务,配送员可通知客户货物已送达,并拍照 发送给客户,方便客户寻找自己的货物。本系统采用 java 技术和 Springboot 搭建系 统框架,系统使用 MySQL 数据库和 Navicat 进行信息管理,设计开发的疫情常态化 下无接触物流配送服务系统。通过调研和分析,系统拥有管理员、用户和配送员三 个角色,主要具备登录注册、个人信息修改、用户管理、配送员管理、物流订单管 理、物流分配管理、物流派送管理、通知信息管理等功能模块。

关键词:疫情常态化 物流配送 无接触送货 Springboot 框架

# **Abstract**

In recent years, with the rapid development of science and technology, the popularity of the Internet has gradually extended to all walks of life, bringing great convenience to people's lives. Under the normal epidemic situation, the contactless logistics distribution service system uses computer network to realize information management, which significantly improves the development and service level of contactless logistics distribution service management under the normal epidemic situation.

This system mainly realizes the contact-free logistics distribution service. The distributor can notify the customer that the goods have been delivered, and send the photos to the customer, so as to facilitate the customer to find their own goods. This system uses java technology and Springboot to build the system framework, the system uses MySQL database and Navicat for information management, and designs and develops a contact-free logistics distribution service system under the normal epidemic situation. Through research and analysis, the system has three roles: administrator, user and deliverer. It mainly has function modules such as login registration, personal information modification, user management, deliverer management, logistics order management, logistics distribution management, logistics delivery management and notification information management.

**Key words:**Normalize the epidemic Logistics distribution Contactless delivery Springboot framework

# 绪 论

当前新型冠状病毒肺炎疫情给我国社会经济发展带来重大影响,居家隔离、 减少外出成为民众新的生活方式,造成线下到店餐饮、住宿等消费被严重抑制, 线上外卖、零售等服务则成为居民新的消费增长点。在消费者普遍担忧交叉感染 的背景下,无接触服务方式,成为保障居民生活的重要手段。在疫情期间,严格 要求骑手全面配备口罩、消毒餐箱等,为用户和骑手提供更进一步的保护。"无 接触配送"服务可以减少面对面接触带来的健康风险,在物流领域有好的应用前 景,满足了在疫情这一特定时期用户对物流服务安全的需要。

在疫情防控期间,为了避免交叉感染,采用无接触物流配送服务,降低因人 际接触增加而导致疫情扩大的风险。还可以提高配送效率,减少等待客户取货的 时间,也方便用户在空闲时间取货。"无接触配送"服务是指用户在通过网络下 单餐饮外卖订单时,通过与骑手协商一个商品放置的指定位置,如公司前台、家 门口等,送达后骑手道通知用户自行取餐。"无接触配送"服务通过减少面对面 接触,保证用户和骑手在收餐环节的安全。但"无接触物流服务",阻断疫情传 播仅仅只是一个开端,真正的意义远不是阻断疫情传播这么简单,将从疫情期间 以防范病毒传播为目的,提高效率,降低成本。向"高效率"、"高安全"的全 面"无接触配送服务发展"。

当前物流配送主要存在几个问题,第一配送服务质量差;第二,配送服务时 间有限,第三,配送路线的选择。路线规划是现代物流配送研究中的关键问题, 研究者们一直在不断优化车辆路径规划问题,分析总结了国内外车辆优化调度问 题的现状,构造模型,求解算法和适用范围。在何玥《校园无人配送优化研究及 系统开发》一文中,提到:意大利学者提出蚁群算法模拟了蚂蚁寻找食物的行为。 蚁群算法具有不错的性能,不仅在路径规划中得到了广泛的应用,而且在许多 其 他领域也得到了广泛的应用。例如指派问题(Assignment Problem)、连续优化 (Continuous Optimization)、调度问题(Scheduling Problem)、图像处理、系 统辨 识等领域[6] 。

随着移动互联网的高速发展,基于Android的手持设备的种类与日俱增,

各类手机 App 的开 发和研制满足了用户各种需要的同时,也加速了社会的进步 和 发展,Android 系统也逐渐成为现今手持设备的主要平台。在徐伟,李挺《基 于 Android 的校园快递配送系统的研究与设计》一文中,使用前后端相结合的应 用,Web 服务器端 使用当下最热门的 Spring Boot 框架结合 Tomcat 服务器以及 My SQL 数据库进行开发,为了方便后期系统的扩展,服务器端 统一返回 json 格式的数据,为客户端提供统一的 Restful api 调 用接口,这样就可以轻松的实 现前后端分离,客户端可以根据 需要进行选择,比如微信小程序、腾讯小程序、 Android 应用等, 为后期客户端程序的扩展预留空间;客户端 App 采用目前主 流 的 Android Studio 作为开发工具,来进行配送系统的设计[4] 。

使用系统的第一步就是进行登录,登录之前需要进行注册,填写账户信息, 如用户名,密码等,注册成功之后,进行登录需要输入用户名和密码,匹配用户 信息表里面的账号密码信息,匹配成功即可登录。

此模块代码包含两个 API 接口:login 和 register。这两个接口上都有 @IgnoreAuth 注解,表示这些接口不需要进行认证即可访问。接口 login 接受三 个参数:username 表示用户名,password 表示密码,captcha 表示验证码。它 检查用户名和密码是否匹配,并如果匹配则生成一个随机的 token 并返回。如果 用户名或密码不正确,则返回一个错误响应。接口 register 接受一个名为 user 的 UsersEntity 对象,该对象包含有用户的基本信息,如用户名、密码、角色等。 它首先使用 MyBatis-Plus 框架中的 EntityWrapper 查询用户是否已存在,如果 存在则返回一个错误响应;如果不存在,则使用 userService 对象执行插入操作, 并返回一个成功响应。在这个接口中,有一个被注释掉的 ValidatorUtils.validateEntity(user) 方法调用,它使用了一个名为 ValidatorUtils 的自定义工具类,用于验证请求体中的参数是否有效。
