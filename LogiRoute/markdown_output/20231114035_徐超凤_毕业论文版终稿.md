## 摘 要

<span id="page-2-0"></span>随着社会的发展,人们对美味健康的食物的需求也日益增长,同时生活节奏的 变快,导致各种以菜谱教学为主要功能的在线学习系统应运而生。在菜谱学习系统 中,如何让用户便捷高效的搜寻既符合口味需求,又满足健康要求的菜谱,是一个 挑战。本课题基于人工智能技术,让用户通过菜品照片直接搜索对应的菜谱,同时, 基于推荐算法,根据用户口味及健康指标,推荐合适的食材,帮助用户快速找到满 意的食谱进行学习。围绕上述核心功能,本课题实现了一个开放式菜谱教学平台, 方便用户发布和分析菜谱教程,进行烹饪方面的沟通交流和菜谱学习。

本课题提出的菜谱学习系统采用 B/S 架构进行开发、前端使用 HTML、后端主 要使用 Java 语言和 SpringBoot 框架,使用 MybatisPlus 实现后端与数据库交互,数 据库采用 MySql。系统包含用户端和管理端,管理端主要包括首页推荐、用户账号 管理、菜谱类型增删查改、图片上传、菜谱标题内容、多少卡路里等功能。用户端 主要包括菜谱首页、菜谱推荐、教学大厅、发布菜谱教学、偏好设置、健康状况管 理等功能。

关键词:人工智能 菜谱学习 菜谱教程 推荐算法

# **Abstract**

<span id="page-3-0"></span>With the development of society, the demand for delicious and healthy food is growing, while the fast-paced lifestyle has led to the emergence of various online learning systems primarily focused on recipe tutorials. In a recipe learning system, enabling users to conveniently and efficiently search for recipes that meet both their taste preferences and health requirements poses a challenge. This project leverages artificial intelligence technology, allowing users to directly search for corresponding recipes based on photos of dishes. Additionally, using recommendation algorithms, it suggests suitable ingredients according to users' taste preferences and health indicators, helping them quickly find satisfactory recipes for learning. Centered around these core functionalities, this project implements an open recipe learning platform, facilitating users to publish and analyze recipe tutorials, engage in culinary communication, and learn recipes.

The proposed recipe learning system adopts a B/S architecture for development. The front end uses HTML, while the back end primarily employs Java and the SpringBoot framework, with MybatisPlus facilitating back-end and database interactions. The database utilizes MySQL. The system consists of a user side and an admin side. The admin side mainly includes functions such as homepage recommendations, user account management, adding/deleting/querying/modifying recipe categories, image uploading, recipe titles and content, and calorie tracking. The user side includes features like a recipe homepage, recipe recommendations, a tutorial hub, publishing recipe tutorials, preference settings, and health condition management.

**Key words** : Artificial Intelligence (AI) Recipe Learning Recipe Tutorial Recommendation Algorithm

## <span id="page-6-1"></span><span id="page-6-0"></span>1.1 研究目的和意义

### <span id="page-6-2"></span>1.1.1 研究目的

随着社会的飞速发展,人们经济水平的提高使人们更加注重自己的生活体验, 尤其是自己日常生活当中的吃喝。现有的菜谱学习方法已经不能满足人们当下的需 求。在互联网的时代背景下,菜谱学习系统逐渐成为人们生活当中学习新菜谱的重 要平台。然而,现有的菜谱学习系统大部分都是在app当中的一部分,没有单独的平 台专门针对菜谱学习。虽然在app当中都可以搜索菜谱和做法,但大部分只有文字描 述并且不统一[\[1\]](#page-58-1)。

菜谱学习系统可以根据菜系还有身体状态精确推荐食谱还有制作方法,帮助用 户快速找到需求方案,解决传统菜谱应用"一刀切"推荐。大大减少缺乏健康关联 性及交互性不足的问题。针对目前菜谱学习的不足,通过增加教学大厅和发布教学 功能,设计了一种基于Spring boot框架和MySQL数据库的内容多样、灵活生动的菜 谱学习系统。使所有用户可以在教学大厅看到所有人发布的的内容并且评论打分, 还可以自由选择分享自己的菜谱。避免了传统学习菜谱的单一性,使智能菜谱学习 推荐系统更加自主化和社会化,打破用户的信息壁垒,适应了新时代人工智能的发 展趋势,有着长远的社会价值,对用户提供了一个新的发展平台,可以通过自身分 享的菜谱吸引顾客,促进饮食文化传播[\[2\]](#page-58-2)。

#### <span id="page-6-3"></span>1.1.2 研究意义

菜谱学习管理系统设计具有多维度创新价值与社会意义。首先,通过分析用户 饮食习惯、健康数据及口味偏好,可以更加精确的推荐用户所需要的菜谱,助力慢 性病管理及科学饮食普及,对于提升公共健康水平具有现实意义。AI驱动的菜谱学 习还能突破传统烹饪边界,集齐世界各地菜谱,既为传统菜系的数字化传承提供新 路径,又促进跨文化饮食创新,推动饮食文化遗产的保护与传播。此外,通过优化 推荐减少了食物浪费[\[3\]](#page-58-3)。该系统还能响应全球可持续发展需求。相较其他成熟的平台, 现有的菜谱学习平台都还不够完善。本系统不仅可以提高用户的学习效率和便捷性。 而且还可以帮助用户之间互相沟通交流。大量减少了人力和时间成本。并且用户有 更多的选择性,大大促进了菜谱的推广。系统还提供了教学大厅功能,使大家可以

互相学习,从而避免因为信息闭塞,而遗失菜谱[\[4\]](#page-58-4)。

## <span id="page-7-0"></span>1.2 国内外研究现状

### <span id="page-7-1"></span>1.2.1 国内研究现状

智能菜谱学习系统是一个便于帮助人们日常生活的平台,伴随着互联网技术的 发展,人工智能菜谱学习也越来越受欢迎。

潘伟杰,冲蕾,李霖奇,王唯佳,施倩倩,陈思枫[5][\(](#page-58-5)2024)基于深度学习与百度 AI 的 菜谱推荐与健康分析微信小程序设计,主要在前作的基础上做出一些改良。利用基 于TensorFlow搭建的神经网络模型Vant组件完善菜品界面;ECharts组件完善卡路里曲 线;利用Python中重量级框架Scrapy和Django,实现搜索关键词、词云图以及其他可视 化图标的功能;同时对小程序的界面进行了一定程度的美化。

蒋东珣[\[6\]](#page-58-6)(2024)高校食堂菜谱推荐系统的数据建模优化设计研究以大数据技术 的飞速发展和算法技术的具体应用为背景,将高校学生每日对食堂菜品选择作为出 发点,分析了高校菜谱推荐系统的研究现状,通过具体的实践和调研,从数据收集、 数据分析和数据建模过程进行了研究设计,在此基础上增加额外因子的影响,对算 法进行进一步的改进和完善,最终给出了适配于不同数据类型的数据处理系统,并 对整套数据分析系统作出有效性讨论。

李志伟[\[7\]](#page-58-7)(2023)基于Eclipse+SDK开发平台的无线点餐系统设计,方便用户点 餐 , 提高点餐效率 , 降低餐饮商家的人工服务成本 , 基于客户机 / 服务器 (Client/Server,C/S)架构和模型-视图-控制器(Model View Controller,MVC)设计思想设 计了一款使用Eclipse开发的无线点餐系统。通过开发工具搭建开发环境,以Java语言 编程实现系统的开发,以可扩展标记语言(eXtensible Markup Language,XML)实现用户 界面(User Interface,UI)的布局,使用MySQL数据库技术和Tomcat轻量级服务器。测试 结果表明,系统设计操作便捷,用户交互性好,提高了餐厅的运营效率和收益。

曹睿[\[8\]](#page-58-8)(2023)基于GBDT的美团菜谱推荐算法研究,根据现有数据研究出了基 于决策提升树的外卖推荐算法。在算法的设计研究过程中,通过对外卖订单数据的初 步分析,根据其特点选取决策提升树作为算法的分类器。在具体模型方面,希望通过使 用3种不同的模型,即XGBoost、CatBoost和LightGBM,来分别得到分类结果并分析研 究在当前环境下性能最强的模型并给出相关建议。希望对于局部数据的研究能够在 一定程度上代表长期的用户外卖订单数据,进而能够为推荐算法的研究提供一定的新 思路。

从国内的菜谱推荐系统来看我国智能餐饮推荐系统的研究呈现技术多元化与场 景细分化特征,深度学习、传统机器学习与轻量化开发框架(如微信小程序)相结 合,推动功能从基础推荐向健康分析、动态可视化等增值服务延伸[\[9\]](#page-58-9)。研究聚焦算法 优化,通过GBDT模型对比(如XGBoost、LightGBM)提升外卖推荐精度,并针对 高校食堂等垂直场景构建多因子数据模型,强化场景适配性[\[10\]](#page-58-10)。当前技术路径覆盖 C/S架构本地化部署与云端协同开发,注重交互体验与运营效率平衡[\[11\]](#page-58-11)。未来发展趋 势将趋向多模态数据融合(如健康监测与饮食记录联动)、跨平台算法泛化及隐私 保护技术应用,但需突破长期用户行为建模、数据孤岛及合规性等瓶颈,推动智能 餐饮系统向实时化、个性化和产业链协同方向发展[\[12\]](#page-58-12)。

#### <span id="page-8-0"></span>1.2.2 国外研究现状

当前,国外菜谱推荐系统的研究聚焦于多模态数据融合与个性化健康导向技术。 通过整合用户饮食偏好、健康指标(如过敏原、卡路里需求)及实时场景,结合深 度学习与知识图谱构建动态推荐模型,使用户拥有更好的体验。

Yu Yang[\[13\]](#page-58-13)(2022)针对疫情下中国社区封闭导致的居民生活不便问题,设计并 实现了一种基于SpringCloud的在线订餐系统。该系统采用前后端分离开发模式:前 端基于Vue.js框架构建用户界面,后端通过SpringCloud微服务架构提供账户管理、菜 单展示、订单处理和用户服务四大核心功能,并利用Feign技术实现服务间负载均衡。 此设计不仅能够满足居民多样化餐饮需求,还能通过精准预测食材用量减少浪费, 同时支持服务范围动态扩展和系统效率优化,为公共卫生事件中的无接触服务提供 了技术解决方案。

近年来,食品智能识别与健康监测技术成为国外学者关注焦点。例如,Abiyev 与 Adepoj[u\[14\]](#page-58-14) 提出了一种融合深度卷积神经网络( CNN )与自注意力机制 (FRCNNSAM)的自动食物识别模型,通过集成异构网络结构、数据增强及正则化 技术,显著缓解了传统模型在复杂场景中的过拟合与欠拟合问题。实验表明,该模 型在Food-101和MA Food-121数据集上分别达到96.40%与95.11%的分类准确率,性能 超越主流迁移学习方法,为精准营养分析与移动端健康管理提供了高鲁棒性技术支 持。此类研究凸显了深度学习在优化食品追踪效率、提升公共卫生服务智能化水平 中的潜力。

邵文娟与刘坤[\[15\]](#page-58-15)(2024)基于SpringBoot框架设计与实现了一套在线点餐系统, 以推动传统餐饮业向信息化与智能化转型。该系统采用Java语言开发,集成 SpringBoot、MyBatis等技术栈,前端通过HTML/JavaScript构建交互界面,后端依托 Apache Shiro和Tomcat服务器实现权限管理与服务部署,数据库采用MySQL支持结构 化数据存储。系统功能涵盖网页设计、账户管理、权限控制及模块化服务,旨在提 升餐饮企业服务效率与管理水平,为核心竞争力构建(如主题餐厅营销策略优化) 提供技术支持,助力行业数字化升级。

# 2 可行性分析和需求分析

## <span id="page-10-1"></span><span id="page-10-0"></span>2.1 可行性分析

#### <span id="page-10-2"></span>2.1.1 市场可行性分析

目前,市场上有很多App中包含菜谱识别系统,比如抖音、百度等软件。并且随 着消费升级,用户对饮食的追求从"吃饱"转向"吃好",更关注健康、营养和个 性化。这促使人们更加追求不一样的菜谱,为需要尝试新口味的用户提供了更高效、 更方便的服务。但这些系统上面的菜谱信息还是过于糅杂,没有办法给用户精确提 供菜谱教学,常常还有其他不相关的内容掺杂在其中。这不仅不方便更不美观,会 导致用户难以找到自己需要的东西,从而浪费时间。线下报班又会大大增加用户的 时间成本和金钱成本,对于想要了解菜谱并且没有时间的用户很不方便。

另外,随着互联网的不断发展,菜谱的不断更新是在所难免的,面对时间和成 本的问题,线上学习菜谱无疑是最好的选择。无论是上班族还是居家都很方便。所 以人工智能的菜谱学习还有很大的发展空间,本系统的开发与实现在市场上是可行 的。

#### <span id="page-10-3"></span>2.1.2 技术可行性分析

本系统采用前后端分离的方式进行开发。前端主要采用HTML、CSS、JS开发 Web页面,后端采用Spring Boot框架Java语言进行开发。

HTML是超文本标记语言,包括一系列标签,通过这些标签可以将网络上的文档 格式统一,使分散的Internet资源连接为一个逻辑整体。HTML文本是由HTML命令组 成的描述性文本,HTML命令可以说明文字,图形、动画、声音、表格、链接等。

CSS是层叠样式表(Cascading Style Sheets)的简称。CSS是一种用来表现HTML 或XML文档的样式的计算机语言。它描述了如何将结构化文档(例如HTML文档或 XML应用)呈现在屏幕、纸质、语音或其他媒体上。

JavaScript(简称JS)是一种广泛应用于Web开发的脚本语言。它主要用于为网 页提供交互功能,如按钮点击、表单验证、动态内容更新等。通过JavaScript,开发 者可以创建更丰富、更动态的用户体验。是一种轻量级、解释型或即时编译型的编 程语言,支持面向对象、命令式和函数式编程风格。

Spring Boot是由Pivotal团队推出的Java开源框架,简化Spring应用的创建、运行、

# 7 研究成果与社会、环境等关系

## <span id="page-55-1"></span><span id="page-55-0"></span>7.1 课题研究成果

本系统主要为了解决在互联网飞速发展的今天,人们没有那么多时间在线下去 专门报班学习菜谱,而且线下学习菜谱也是固定的,没有灵活选择。也不清楚这个 菜谱类型是否适合自己,增加了线下学习的时间和金钱成本。在本次课题研究中, 首先根据用户偏好设置和健康历史智能推荐适合的菜谱,使用户大大减少时间成本 还能选择自己最适合的菜谱菜系。

从系统开发与用户的角度进行分析,智能推荐菜谱学习系统无疑帮助用户选择 了更加安全的饮食等问题。用户可以随时查看菜谱教程和发布菜谱教程,分享自己 的心得,帮助大家学习更多菜系。

## <span id="page-55-2"></span>7.2 研究成果与社会的发展

关于人们的日常生活,大家也越来越重视自己的生活品质,这变相的促进了AI 菜谱的发展。而AI智能推荐菜谱也是顺应时代研究而出现,人工智能菜谱学习系统 作为食品科技与AI技术的融合创新,正在深刻改变人类的饮食方式和食品产业格局。 在技术层面,最新研究实现了三大突破:基于深度学习的多模态菜谱生成系统可同 时处理文字、图像和视频数据,这些技术创新带来了显著的社会效益。

然而,技术发展也面临文化同质化和数据隐私等挑战。未来发展方向包括建立 多元文化数据库、开发可解释性AI烹饪决策系统,以及构建食品评估框架。人工智 能菜谱系统正从技术工具演变为连接饮食文化、健康管理和可持续发展的关键节点, 其发展将深刻影响人们未来的生活方式。

## 结论

<span id="page-56-0"></span>本课题开发的系统为人工智能推荐菜谱学习平台,系统总体分为用户端和管理 端,采用管理端和用户端分离的方式开发。主要是为了解决市面上系统的菜谱信息 还是不够完善的问题,用户在浏览推荐的菜谱时,往往推荐的都是不符合用户需求 的,常常还有其他不相关的内容掺杂在其中,没有办法给用户好的体验。这不仅不 方便更不美观,会导致用户难以找到自己需要的东西,从而浪费时间。线下报班又 会大大增加用户的时间成本和金钱成本,对于想要了解菜谱并且没有时间的用户很 不方便。本系统就可以很好的规避这些问题。

系统用户端中包含了登录、注册、偏好设置、健康状况管理、健康提交历史、 菜谱首页 、菜谱推荐、发布教学、教学大厅等功能。以上功能基本满足用户的需求, 用户可以在该系统中填写自己偏好还有健康状态,菜谱首页会根据用户填写的情况 采用推荐算法给用户推荐适合的菜谱并附带教学。除此之外,该系统还提供了教学 大厅功能,为用户提供一个学习分享的平台,用户还可以根据自己想要分享的菜谱 去选择菜系、图片,发布以后其他用户还可以在下面评论、评分,大大提高了交流 能力,促进了用户的积极性。

系统管理端中包含了登录、菜谱类型管理、口味管理、菜谱管理等功能。这些 功能就是为了更好的帮助用户去找到自己需要的菜谱,及时更新菜谱类型、口味, 并维护系统环境,为用户提供一个良好学习菜谱的平台。

该系统严格按照软件开发流程,并对系统功能进行了手动测试,确保系统能够 平稳运行。系统虽然已经开发完成,但还是存在一些缺陷,在数据库中菜谱信息还 是较少。后续也会不断完善系统,避免因为这些缺陷而给用户带来不好的体验。菜 谱学习系统是有很大发展空间的,尤其是随着人们生活水平不断提高对菜谱的需求 也在不断提高。
