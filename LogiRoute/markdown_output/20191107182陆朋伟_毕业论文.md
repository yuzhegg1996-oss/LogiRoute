# 摘 要

<span id="page-2-0"></span>高校宿舍管理系统是一种能够帮助大学宿舍管理人员更好地管理和维护学生宿舍的软 件系统。该系统主要包含以下几个方面:

首先,宿舍管理系统可以维护宿舍基本信息。对宿舍的房间号,所在楼层,房间类型等 信息进行标注和管理。在学生入住时,可通过系统进行预分配,并对床铺信息、入住时间、 房间设备情况等进行记录和管理;

其次,宿舍管理系统可以帮助管理人员对入住学生信息进行统一管理。学生登录后需注 册账号,并填写学号,姓名,性别,年龄,所在寝室,手机号,身份证等信息。而拥有权 限的管理员可对以上信息进行查看和修改。

再次,宿舍管理系统能够帮助管理人员进行安全监管。例如,在疫情期间,系统能够及 时掌握学生健康状态、实现每日管理,并对来访人员实施应急管控;宿舍管理系统在疫情 管控期间还支持楼层封控管理。对突发疫情的楼层进行封控标记,直到疫情结束再将其解 除封控状态。

该系统的编程实现为前端采用 HTML、CSS、JavaScript 等核心技术,使用 vue.js 框架 并结合 ElementUI 组件库。后端使用的编程语言为 Java,基于 Tomcat + MySQL+SSM 框架, 在 IntelliJ IDEA 开发平台进行设计实现。

整个系统采用 MVC 模式(M(Model:模型层)、V(View:表现层)、C(Controller: 控制层))三层架构模式来实现了整个程序架构的分析、设计及具体实现。

管理人员通过本系统模块可以实时管理学生信息,如:录入新生、查询或更新学生以 及房间的信息、查询学生体温和楼层信息、来访人员登记、学生行程申报及管理员审核、 系统设置等功能。该系统具有良好的可操作性、可维护性、可扩展性和可移植性,能可靠 高效地完成用户的需求。

关键词:疫情防控 宿舍管理 Vue SSM

# **Abstract**

<span id="page-3-0"></span>College dormitory management system is a kind of software system that can help college dormitory administrators manage and maintain students' dormitories better. The system mainly includes the following aspects:

First of all, the dormitory management system can maintain the basic information of the dormitory. Label and manage dormitory room number, floor, room type and other information. When students check in, they can be pre-allocated through the system, and record and manage the bed information, check-in time, room equipment, etc.

Thirdly, dormitory management system can help management personnel to carry out safety supervision. For example, during the epidemic, the system can timely grasp the health status of students, realize daily management, and implement emergency control for visitors;The dormitory management system also supports floor containment management during the epidemic control period. The floor of the outbreak shall be sealed and marked until the outbreak is over.

The programming of the system adopts HTML, CSS, JavaScript and other core technologies in the front end, and uses UE.js framework combined with ElementUI component library. The back-end programming language is Java, based on TOMCAT + MySQL+SSM framework, and designed and implemented on IntelliJ IDEA development platform.

The whole system adopts MVC pattern (M (Model: Model layer), V (View: presentation layer), C (Controller: control layer)) three-layer architecture pattern to realize the analysis, design and concrete implementation of the entire program architecture.

Administrators can manage student information in real time through this system module, such as: input new students, query or update the information of students and rooms, query student body temperature and floor information, visitor registration, students going out application and administrator review, system Settings and other functions. The system has good operability, maintainability, expansibility and portability, and can meet the needs of users reliably and efficiently.

# 1 绪 论

## <span id="page-6-1"></span><span id="page-6-0"></span>1.1 系统设计的目的和意义

在高校日常管理中,学生宿舍管理是一项至关重要的工作。有效的宿舍管理不仅影 响学校的整体管理水平,也直接反映出学校口碑和教学质量。然而,目前许多学校仍然 采用传统的手工记录方式,导致数据量庞大、效率低下。特别是新冠疫情的爆发,增加 了高校宿舍管理的难度和挑战,也使得需要一款融合了功能全面、操作简便、防疫措施 的宿舍管理软件进行支持。只有通过引入先进的管理软件,才能提高宿舍管理及效率, 并更好地服务于学校日常工作[1]。

## <span id="page-6-2"></span>1.2 国内外研究现状

在当今迅速发展的社会中,计算机化管理已成为高校日常管理系统中不可或缺的重 要组成部分。随着技术快速变革和信息化水平的提升,传统的手动记录方式势必将被淘 汰。因此,应用计算机技术对于高校日常宿舍管理的完善和现代化至关重要。我国自 70 年代末开始逐渐推广计算机在管理领域的应用,但与其他发达国家的应用水平相比仍有 较大差距。但近年来,在政府支持下,高校学生宿舍管理系统由原始的手工记录向计算 机化管理转型,得到了长足发展。这种转型不仅加速了高校的教育管理信息化进程,还 提高了高校管理服务水平,以及管理人员的工作效率。可以肯定地说,高校学生宿舍管 理系统通过计算机技术的崭新运用,实现了教育管理信息化改革,使学生宿舍管理更加 规范化、高效化和便捷化[2]。同时,它也为高校日常管理系统的现代化建设提供了新的思 路和方式,推动了整个高校管理系统的改革创新。

国外高校通常拥有大型且稳定的技术团队,专门提供服务和技术支持。而相比之下, 国内高校在信息化建设方面较为滞后。在数字化校园的逐步推广过程中,各高校不断投 资于硬件、系统软件和网络设施的建设,并开发不同应用系统来支持教学、科研及办公 管理等活动。这些工作逐渐构建起一定规模的信息化建设体系。

然而由于整体信息化水平相对落后、经费紧缺以及理论体系不完善等因素,国内高 校的宿舍管理系统与国外高校相比在机构设置、服务范围、服务质量、人员要求等方面 存在一定差距。因此,要使国内高校的宿舍管理系统不断进步,需要进一步加大投资力 度,促进信息化技术应用的普及并借鉴先进的管理理念和技术经验[3]。可以预见的是,在 未来的日子里,随着信息化建设不断发展,国内高校的宿舍管理系统将不断得到完善和 升级。

通过比较国内外宿舍管理系统的研究现状可以发现,在安全性和信息更新方面,与 那些发达国家相比,国内宿舍管理系统还存在一定的差距。目前各个系统的数据基本都 是单独建立的,共享性较差。此外,由于信息主要由管理者为主体进行采集、整理及更 新,其取舍往往依赖于管理者对于信息的认知和喜好。这导致难以很好地掌握用户的真 实需求,难以满足用户的需要。

为了解决这些问题,传统的宿舍管理应该朝着数字化、无纸化、智能化和综合化的 方向发展。这将为进一步完善计算机宿舍管理系统打下良好的基础。此举可以提高信息 的更新速度和安全性,同时也可以更好地满足用户需求,推动宿舍管理工作不断创新和 发展[4]。

# 2 系统分析和总体规划

## <span id="page-8-1"></span><span id="page-8-0"></span>2.1 所需系统的理由

#### 1.必要性

高校的日常管理工作中,学生宿舍管理是重要的组成部分之一,管理水平的高低关 系到整个学校的管理水平和教学质量。然而,许多学校在宿舍管理方面仍然依靠人工记 录数据,这种方法在数据量大、需要频繁查找记录时十分不便,工作效率也较低。针对 这个问题,开发一款系统化的宿舍管理软件变得尤为必要。特别是在新冠疫情期间,高 校宿舍区域的管理更加重要。因此,这款软件需要包含完备的功能,操作简单易学,并 且能够兼顾防疫措施的实施。这样不仅有利于提升学校管理和教学质量,还能有效保障 全体师生的生命安全和身体健康。

#### 2.可行性

- (1)经济可行性:该系统是完全由个人开发完成,系统的开发不需要耗费过多的人 力和财力,所以在经济层面是完全可行的。
- (2)操作可行性:目前,计算机的普及已经覆盖了绝大多数单位和家庭,且该系统 的功能和界面并不复杂。只需使用相应的用户帐户和密码,相关人员就能够进入系统,方 便地对信息进行管理。因此其操作是完全可行的。
- (3)技术可行性:本系统前端采用常用的 HTML+CSS+JavaScript,结合 Vue 框架, 后端采用 Java 语言,基于 Tomcat + MySQL+SSM 框架,开发者根据自身所学知识并结合 网上现有资料完全有能力开发此系统。所以技术层面也是可行的。

## 2.2 需求分析

- 1.数据需求:本系统主要完成学生入住登记及入住后的管理。因此,需要的数据信息 包括管理员、学生、宿舍等信息。
  - (1)管理员信息:包括个人帐号和密码等信息;
- (2)学生信息:包括学号、姓名、性别、年龄、所在宿舍号、手机号、身份证号等信 息;
  - (3)宿舍信息:包括宿舍号、所在楼层号、房间类型、房间设备等信息;
  - (4)楼层信息:包含楼宇名称、楼层数、封控状态等信息;
  - (5)来访者信息:包括来访者姓名、电话及进入和离开时间等信息。

- 2.功能需求:进行功能需求分析的目的是了解用户对于目标系统数据处理功能的需 求,即明确相关功能方面的要求和期望。本系统的功能需求有:
  - (1)管理人员和学生通过输入帐号和密码,验证通过后进入系统;
  - (2)管理人员可以随时录入某个学生或房间的住宿情况;
  - (3)管理可以随时查询某个学生或房间的住宿情况;
  - (4)管理人员在疫情环境下设置宿舍楼层或房间进入封闭状态;
  - (5)学生在疫情环境下可以上传自己的每日体温;
  - (6)学生在疫情环境下外出可以在系统上向管理人员提出申请;
  - (7)管理人员审核学生发出的外出申请;
  - (8)管理人员可以记录来访者的基本信息和出入情况。

## <span id="page-9-0"></span>2.3 总体设计

#### <span id="page-9-1"></span>2.3.1 功能模块设计

- 1.功能定义
- (1)个人中心:管理人员和学生在此模块可对个人基本信息进行查看和修改;
- (2)体温信息管理:对于管理人员,此模块可对学生上传的体温信息进行统一记录并 对体温异常的学生进行特殊标注;对于学生,在此模块上传自己的每日体温;
- (3)行程申报管理:对于管理人员,此模块用于对学生发出的外出申请进行审批;对 于学生,在此模块填写本人外出的原因及去向等信息后向管理人员发出申请;
  - (4)来访信息管理:管理人员对来访者信息进行记录;
  - (5) 楼层信息管理:管理人员对楼层信息进行维护;
  - (6)楼层封控管理:管理人员设置楼层封控状态;
  - (7) 公告管理:管理人员可发出相关公告并维护,学生则只拥有查看的权限。

## 4 系统测试

## <span id="page-29-1"></span><span id="page-29-0"></span>4.1 软件测试的目的和原则

在大型软件系统开发过程中,我们会遇到各种错综复杂的问题,因此,每个阶段结 束之前进行严格的技术审查以尽早发现和纠正错误非常必要。软件测试目的在于在软件 产品投入生产性运行之前尽量多地发现其中可能存在的错误。在当前阶段,软件测试仍 然是保证软件质量的关键步骤之一,通过测试来对软件规格说明、设计和编码进行最后 复审[8]。

尽管面向对象技术的基本理念已经提高了软件质量,但不可避免地,在任何编程技术 中,程序员产生错误是常态。在面向对象开发中,由于代码重用率高,测试显得更加重要 以避免错误不断积累。所以在面向对象编程范式下,软件测试更具有关键性。

与开发过程类似,测试过程也需要逐步分阶段进行,每个步骤都建立在前一个步骤之 上。大型软件系统通常由许多子系统和模块组成,因此,对大型软件系统的测试基本上有 以下层级结构,如图 4-1 所示。

![](_page_29_Figure_5.jpeg)

本次针对该系统重点进行了功能测试。所谓功能测试,即对系统各个页面进行功能验 证的过程。通过执行功能测试用例,逐一检查产品的各项功能是否正常,并能否满足客户 的需求。虽然相比于其他类型的测试,功能测试涉及的业务逻辑较为简单,但是做好功能 测试同样需要遵循完整的测试流程和方法[9]。在测试过程中,要对系统功能的正确性、完 整性、可靠性等方面进行评估,并通过测试数据、异常数据等多种方式,对系统的功能进 行全面覆盖,以验证其性能和稳定性。

通过执行测试用例,可以检查每个功能是否按照预期运行,并记录测试结果,以便开

发人员改进和修复程序中存在的缺陷。

# <span id="page-30-0"></span>4.2 设计与编写测试用例

登录功能是一个软件系统中非常重要的组成部分,它对于保护系统数据和用户账户的 安全至关重要。通过登录功能,用户可以输入正确的用户名和密码来验证自己的身份,从 而获取访问系统的权限,不允许未经授权的用户访问敏感信息,防止恶意攻击、数据泄露 等安全风险[10]。另外,登录功能也有利于系统的管理和维护。通过记录用户的登录信息, 可以方便管理员对用户行为进行跟踪和管理,确保各项操作的合法性和规范性。因此,登 录功能在现代软件开发中是不可或缺的一部分,它对于系统的安全性、稳定性和可靠性都 有着非常重要的影响,需要进行功能测试。

## <span id="page-36-0"></span>4.3 测试方法

本系统在功能测试中主要使用到场景法以及边界值分析法等常用的测试用例设计方 法。

#### (1)场景法:

模拟用户操作软件时的场景,主要用于测试系统的业务流。分为基本流和备选流。基 本流按照正确的操作流程来测,备选流则是按照错误流程来测。例如本系统的登录功能就 是采用该方法来完成测试。登录功能的基本流是输入账号、密码—>选择登录身份—>验证 账号、密码以及身份是否正确—>登录成功,进入首页。备选流是账号不存在/密码错误/账 号与身份不匹配—>返回错误提示并停留在登录页面。因此基于上述分析设计出了登录功 能的测试用例。

#### (2)边界值分析法:

边界值分析法是对输入或者输出的边界值进行测试。首先通过需求分析,找出边界, 然后再写出小于最小值以及大于最大值的两个数。例如本系统体温信息管理中的预警功能 测试,正常体温范围是 36.5-37.2 摄氏度,边界值为 36.5 和 37.2,所以在测试时不仅需要 输入正常范围内的数据,还需输入小于 36.5 以及大于 37.2 的测试数据用来测试预警功能 是否正常。

# 5 研究成果与社会关系

随着新冠肺炎疫情的全球爆发,各个国家都在加紧部署抗疫工作。宿舍作为一个人 们生活、学习、工作的必要场所,在疫情防控中起到了重要的作用。因此,开展基于疫 情防控下的宿舍管理系统研究对防止疫情扩散、保障人民群众身体健康和生命安全具有 重要意义。

首先,在疫情防控下,开展宿舍管理系统研究对于提高宿舍管理工作效率和精度具 有积极作用。疫情期间,宿舍管理涉及到大量的信息的统计、分析和反馈工作,需要依 赖于科学的信息化手段,如智能化监管系统、数据管理平台等。通过这些先进技术的应 用,可以有效降低管理成本,提高管理效率和准确性,同时也更好地保证了宿舍管理工 作的公正、透明和权威性[15]。

其次,在疫情防控下,开展宿舍管理系统研究还对于促进和谐社会关系具有重要意 义。在实际操作中,需要充分考虑住户的个人隐私和通行自由等关键问题,并针对不同 的管理对象,采取差异化的管理措施,让住户感受到公平、公正、公开的宿舍管理。同 时,也需要提高住户参与管理的积极性和主动性,在管理中贯彻共建共享、和谐共处的 理念。这些努力将进一步加强社会稳定和谐的基础,提升人民生活品质和幸福指数。

最后,在疫情防控下,开展宿舍管理系统研究还能够为国家部署集中式隔离医学观 察中提供有益的借鉴。以专业化医学观察站为例,它的设备配置、服务标准、管理模式 等都岗位了高要求。而基于先进的宿舍管理系统进行管理,无疑是优化医学观察站管理 的重要手段之一。未来,集中隔离医学观察工作还将长期延续,这就需要我们深入研究, 综合利用好现有的管理技术,从而更好地应对类似的突发事件,为国家护航安全保障和 公共卫生事业做出更大贡献。

综上所述,基于疫情防控下的宿舍管理系统的研究成果对推进数字化、信息化、智 能化的宿舍管理工作,提高社会关系和谐稳定等方面都发挥了重要作用。这也凸显了科 技创新在现代社会建设中的不可替代作用和广泛应用前景。

- (1)采用 MVC 三层体系结构,使系统具有很好的可维护性、可重用性和可扩展性。

  - (3)页面简洁明了,用户可以很容易地进行操作。

纵观来说,该系统规模不大,由个人独立完成分析、设计、开发和测试,满足了预期 的功能需求,但仍然有一些方面可以进一步改进和完善。例如系统整体界面不够精美,配 色比较单调,可以适当丰富系统界面的色彩。其次还可扩充一些同样常用的功能,诸如物 品损坏上报、维修等功能。
