**Design and Implementation of an Image Recognition-Based Financial Reimbursement System**

## 摘 要

<span id="page-2-0"></span>随着企业数字化转型的加速,票据处理已成为现代财务管理的关键环节。在快 节奏的商业环境下,传统的手工报销流程存在效率低下、人工核验耗时、票据真伪 难辨等缺点,尤其是当面对大量不同格式票据时,容易出现错审漏审风险。相较于 传统报销流程,本系统优化了人工报销大部分缺点,实现了全自动报销流程,为企 业提供了快捷可靠的报销方式。

基于图像识别的财务报销系统在开发过程中采用了分层架构的设计原则和前后 端分离的开发模式,集成了图像识别与自然语言处理技术,构建了包含用户管理、 票据识别、数据分析、统计报表、智能问答、智能审核等六大功能模块。系统前端 采用 Vue3 框架构建响应式界面,后端基于 Node.js 开发,利用 Express 框架搭建 RESTful API 服务,在数据存储方面,财务报销系统采用 Pinecone 向量数据库和 MongoDB 数据库进行数据存储,使用 RAG 技术将报销内容转化为向量表示,并存 储在向量数据库中,结合 Pinecone 向量数据库实现查找内容的精准检索。在关键技 术实现方面通过 GPT-4o API 解析票据图像,自动提取关键字段并进行智能审核后完 成表单预填。

关键词:图像识别 财务报销 RAG

## **Abstract**

<span id="page-3-0"></span>With the rapid development of digital transformation in domestic enterprises, invoice processing has become a critical component of financial management. In fast-paced business environments, traditional manual reimbursement workflows suffer from inefficiencies and time-consuming verification processes, particularly when handling large volumes of invoices in diverse formats, which increases risks of erroneous or missed audits. Compared to conventional reimbursement methods, this system addresses most shortcomings of manual processing by implementing fully automated workflows, providing enterprises with efficient and reliable reimbursement solutions.

The financial reimbursement system based on image recognition adopts a layered architecture design and front-end/back-end separation development model during its development process. It integrates image recognition and natural language processing technologies, constructing six major functional modules: user management, receipt recognition, data analysis, statistical reporting, intelligent Q&A, and intelligent auditing. The system's front-end uses the Vue3 framework to build a responsive interface, while the back-end is developed based on Node.js, utilizing the Express framework to set up RESTful API services. For data storage, the financial reimbursement system employs the Pinecone vector database and MongoDB database, using RAG technology to convert reimbursement content into vector representations stored in the vector database. In terms of key technology implementation, the system uses the GPT-4o API to parse receipt images, automatically extract key fields, and perform intelligent auditing before pre-filling the forms.

**Key words**:image recognition automatic reimbursement RAG

## 1 绪 论

## <span id="page-6-1"></span><span id="page-6-0"></span>1.1 研究目的和意义

#### <span id="page-6-2"></span>1.1.1 研究目的

近些年,随着国内企业的快速发展,财务报销作为财务管理的核心环节,其处 理效率直接影响到企业管理效能[\[1\]](#page-37-1)。传统报销流程普遍存在审批周期长、票据核验准 确率低、合规检查依赖人工等问题[\[2\]](#page-37-2)。根据行业报告显示,大部分企业会因为报销单 据填写不规范导致返工,大量财务人员每周会耗费数小时以上处理票据核验[\[3\]](#page-37-3)。同时 审核过程中会存在虚假报销、预算超支、敏感消费等问题,票据识别成为企业财务 管理的重点[\[4\]](#page-37-4)。

图像识别技术在这两年内发展迅速,当下许多热门软件都有应用到这个技术。 比如淘宝、京东等购物平台,用户可以将商品的照片上传并识别,系统会给提供相 似的商品。这个技术同样可以应用到报销系统中,传统的报销流程每一步都需要人 工操作,报销流程复杂且繁琐,加入图像识别功能后可以极大减少人工成本,提高 工作效率,并且依赖于精准的图像识别,在智能报销系统中很难出现漏审或错审等 问题。

#### <span id="page-6-3"></span>1.1.2 研究意义

基于图像识别的财务报销系统设计具有重要现实意义。和国外成熟的财务报销 管理系统的对比中,我国企业的报销流程存在纸质单据处理效率低和人工审核易出 错等诸多问题[\[5\]](#page-37-5)。如何利用AI技术突破传统报销模式的限制,开发出能够精准处理多 种票据数据,并且能够动态适应复杂审核规则的解决方案是需要深入研究的核心问 题,探索具有实际价值的财务报销系统尤为重要[\[6\]](#page-37-6)。基于图像识别的财务报销系统通 过图像识别技术实现票据信息的自动提取,可以有效的解决传统报销中人工录入的 缺点。用户通过手动上传即可完成多种格式票据的快速上传,而且能够实时追踪审 批进度,非常显著的提升了财务处理效率。目前研究主要是针对复杂票面信息的图 片识别准确率,特别是针对手写体和模糊图像等特殊场景的识别效果[\[7\]](#page-37-7)。针对敏感报 销,系统还可实时比对报销标准库,标记异常支出,并对用户进行提醒,在简化操 作流程的同时提高了合规性。

## <span id="page-7-0"></span>1.2 国内外研究现状

### <span id="page-7-1"></span>1.2.1 国内研究现状

国内企业目前的报销流程依赖人工操作,并且相关研究非常少,但财务报销作 为企业日常运营的重要步骤,智能化报销系统的出现尤为重要。随着计算机视觉技 术的发展,基于图像识别的财务报销系统也随之发展起来。

夏可青(2024)打通各部门的信息传导,整合财务系统与业务系统以建立业财融 合的全流程线上报销平台,实现智能财务预测与决策[\[8\]](#page-37-8)。

谷丛丛(2025)通过引入先进的信息技术来优化报销流程,提高了财务管理的 规范性,并且重点研究了财务信息化报销系统的需求,为同类系统的建设和改进提 供了参考[\[9\]](#page-37-9)。

徐智芳(2025)通过对费用报销流程的深入分析,为企业在智能化转型中提供 有效的解决方案,从而实现精准高效的财务管理[\[10\]](#page-37-10)。

罗宏(2021)提出基于图像识别算法的财务凭证自动录入预处理系统。针对我 国大多数集团企业及其子公司财务较为独立的特点,在进行系统需求分析基础上,进 行了图像识别和自动录入系统的设计,最后基于GA算法对该财务系统进行了改进并 对比分析[\[11\]](#page-37-11)。

刘雪颖(2025)阐述了RAG的基础概念以及工作流程,细致地梳理了此项技术 的当前状况,剖析了现有RAG技术所有的优势与存在的不足[\[12\]](#page-37-12)。

国内在图像识别财务报销系统研究领域取得了一定成果,研究主要围绕智能化 技术与传统财务流程的融合展开,当前研究借助光学字符识别也就是OCR技术实现 票据关键信息的自动化提取与结构化处理,凭借运用深度学习算法提高识别准确率, 同时集成税务系统以完成票据真伪核验和重复报销检测。部分学者对多模态技术的 应用进行了探索,例如结合自然语言处理来解析票据文字描述,利用区块链技术保 证数据不可篡改,构建端到端的可信报销流程,在系统架构方面,研究重视模块化 设计与分布式部署,形成线上线下协同的无接触式报销模式。部分研究成果已从技 术验证走向实际应用,比如高校和医疗机构构建业财融合平台,实现跨部门数据共 享与智能分析,未来研究趋势将朝着多技术融合方向发展,包括结合物联网实时采

集业务数据优化智能审核规则,以及借助云原生架构提升系统扩展性。

#### <span id="page-8-0"></span>1.2.2 国外研究现状

在全国都在推进数字化转型的当前形势下,借助图像识别技术构建的财务报销 系统已然成为了国内外学术界关注的焦点,欧美等发达国家凭借自身的技术积累以 及政策方面的支持,在技术创新应用和数据安全等领域走出了有差异的研究道路。

Edward Sun(2025)提出了跨模态票据分析框架,此框架借助将视觉特征与语 义信息相互融合,在对医疗账单、国际差旅票据等复杂文档进行识别时,实现了98% 的字段提取准确率,很好地化解了跨国企业财务数据隐私保护和模型训练之间存在 的矛盾[\[13\]](#page-37-13)。

David Kainer(2025)描述了一系列工作流程,这些工作流程结合了 OpenAI GPT 的功能,以使用本体术语注释拟南芥和森林树木表型观察,旨在获得类似于手动策 划注释的结果[\[14\]](#page-37-14)。

Abràmoff Michael D(2022)针对自动报销不合理的问题,提出了一种基于 k-means++聚类和多元线性回归的预测定价方法,并对模型的低准确率进行了评价[\[15\]](#page-37-14)。

国外系统早已将AI技术应用到财务报销上,实现了报销流程的自动化,大大节 约了人力成本,但国外在隐私保护和复杂票据识别反面仍有许多要改进的地方。

#### <span id="page-8-1"></span>1.2.3 现状总结

图像识别技术下的财务报销系统,其国内外发展路径存在差异,国内研究重点 在于流程自动化以及智能化的升级,利用OCR技术可自动抓取发票信息,同时还可 以对发票真伪进行校验,达成从拍照上传直至资金支付的全链条无纸化操作,智能 审核模块引入了机器学习算法,可自动将费用标准和预算规则进行匹配,对于异常 单据会实时发出预警,部分高校借助区块链技术构建电子票据溯源机制,以此保证 数据不会被篡改。国外研究更侧重于复杂场景中的技术创新与合规保障,跨模态识 别框架可有效解析像医疗账单这类非标票据,平衡数据隐私与模型训练的需求。

## 2 系统分析

## <span id="page-9-1"></span><span id="page-9-0"></span>2.1 可行性分析

#### <span id="page-9-2"></span>2.1.1 市场可行性分析

在当前市场上,财务报销系统大部分还是通过人工审核。但人工审核存在费时 费力、错审漏审等许多缺点,企业为解决这一问题,需要进行进一步人工审核,这 导致了企业报销周期长和人力成本高,企业对自动化报销系统的需求进一步增加。 而智能报销系统完全解决了这些问题,通过精准的图像识别使报销单的内容出现错 误的概率大大降低,并且系统支持自动审核,减少了人力成本。所以基于图像识别 的财务报销系统的实现在商场上是完全具备可行性的。

#### <span id="page-9-3"></span>2.1.2 技术可行性分析

系统采用前后端分离架构,前端基于Vue 3框架构建,Element Plus组件库提供标 准化UI控件,简化了前端编写难度。后端采用Node.js与Express框架,数据库选用 MongoDB,用来存储如用户信息、票据图像等数据。在图像识别方面,OpenAI的 GPT-4o模型可以对票据进行识别,Pinecone向量数据库的近似最近邻检索算法可以 极大减少数据查询时间,提高了智能助手响应效率。以上技术都是目前市面上主流 的开源技术,同时前后端分离开发模式简化了开发难度,所以基于图像识别的财务 报销系统的实现在技术上是完全具备可行性的。

#### <span id="page-9-4"></span>2.1.3 经济可行性分析

开发基于图像识别的财务报销系统所用到的技术大部分为开源产品,没有软件 授权费用,只有GPT4O API需要收费,但每次问答和图像识别的费用极低。开发工 具采用Visual Studio Code与MongoDB等免费工具,同时开发阶段和生产阶段均由我 个人完成,又因为前后端分离的特性,所以开发和维护的成本非常低,并且此项目 功能需求非常明确,不需要进行多余设计。基于以上分析,本系统的开发与实现在 经济上是可行的。

## <span id="page-9-5"></span>2.2 需求分析

基于图像识别的财务报销系统主要面向两类用户群体,一是中小企业财务部门, 需要高频处理小规模报销任务问题,二是大型企业员工,财务部门急需简化多类型

票据的提交与审核流程。此项目针对企业进行了优化和调整,能够一站式解决企业 报销流程,还能让企业员工通过智能助手快速查找到需要的内容,极大提高了企业 员工的工作效率。基于以上分析,本系统的开发与实现在需求上是可行的。

## <span id="page-10-0"></span>2.3 相关技术

#### <span id="page-10-1"></span>2.3.1 前后端分离开发模式

基于图像识别的财务报销系统目前采用前后端分离架构,前端基于Vue 3框架构 建,通过Axios与后端Express API进行异步通信。后端部分基于 Node.js 开发,并利 用 Express 框架搭建了 RESTful API 服务,在开发阶段,预先设置数据交互接口, 以实现并行开发和测试,后端不再控制前端渲染的效果,而是仅按照提前设置好的 数据接口以JSON格式向前端提供可调用的API服务。在用户认证模块中,前端通过 Vue Router控制权限路由跳转,后端通过JWT生成加密令牌并校验角色权限,确保管 理员与员工功能隔离。

#### <span id="page-10-2"></span>2.3.2 前端技术体系

前端采用Vue 3组合式API实现组件化开发,结合Element Plus组件库提供表单表 格及图表组件,Element Plus能够快速构建系统管理页面,例如报销申请页通过 ElUpload组件实现收据图片上传,仪表盘页通过ECharts渲染柱状图与饼图。Vuex集 中管理用户身份和报销限额等全局状态,结合Vue Router导航守卫实现动态权限控 制,如检测到员工角色访问管理员统计页面时自动跳转至首页。

#### <span id="page-10-3"></span>2.3.3 后端技术体系

后端基于Node.js与Express框架搭建服务,通过Express快速构建RESTful API,提 供理由中间件和请求处理等功能,采用MongoDB和Pinecone向量数据库存储报销记 录文档型数据,利用Mongoose定义数据模型并实现工号自增和月度限额校验等业务 规则。并且集成了GPT-4oAPI实现收据内容提取与自然语言查询,GPT API能够精准 提取票据中的数据并返回结果,结合数据库中的内容还能生成对应报告,例如上传 收据图片后调用AI接口提取金额、销售方信息,结合Pinecone向量数据库实现RAG 技术优化智能助手。

#### <span id="page-10-4"></span>2.3.4 数据库设计

系统采用MongoDB数据库和Pinecone向量数据库作为主数据库,MongoDB是一

个NoSQL数据库,采用文档类型存储,是目前比较流行的非关系型数据库。MongoDB 存储数据的格式为JSON,要比数据库Mysql更好用,利用其灵活的模式设计存储动 态报销数据,例如报销记录集合包含收据图片路径、AI解析文本及多级审核状态。 Pinecone 是一个云原生向量数据库,专为存储和检索高维向量数据设计,主要用于 支持机器学习和搜索功能。系统使用 Pinecone 配合 RAG 技术存储报销内容的向量 表示。可以通过自然语言从向量数据库中提取数据然后交给GPT分析并生成报告, 很大程度提高了系统的响应效率。

#### <span id="page-11-0"></span>2.3.5 智能化技术集成

系统深度集成GPT-4o多模态模型,借助GPT-4o多模态模型与RAG技术的深度融 合,达成了高效的票据识别、合规性检查、数据分析以及智能问答等功能,为企业 财务报销给予了智能化的解决方案,图片识别模块会把图片转化成Base64编码, GPT-4o可同时处理图像与文本输入,系统经过精心设计的提示来引导模型提取关键 字段,接着调用GPT API提取数据开展合规性检查,借助语义分析检测违规关键词并 触发预警。智能助手功能与RAG技术相结合,利用Pinecone存储历史报销数据的向量 化表示,在用户提问时先检索相似记录再发送给GPT生成相应的回复,智能助手功 能是系统智能化集成的又一关键表现,运用了RAG技术来提高回答的精准性与上下 文相关性,系统达成了从票据识别到智能问答的全流程自动化,提升了财务管理的 效率和准确性,还为用户提供了便捷的交互体验。系统的统计分析模块也融入了智 能化技术,系统凭借MongoDB聚合查询获取指定月份的报销数据,并将其格式化为 文本,结合提示发送至GPT-4o模型,生成包含总金额、部门分布、异常趋势等内容 的月度分析报告,这种基于AI的分析能力可管理员快速查看财务状况,优化资源分 配。

- (2)报销列表:用户可随时查看自己提交的报销申请的实时状态,系统将申请 状态清晰地划分成"待审核"、"已批准"以及"已拒绝"这三种类型,便于用户 迅速了解进展状况,对于被拒绝的申请,系统会明确显示管理员给出的具体拒绝原 因,帮助用户知晓问题所在,在下次提交时可进行改进。

(4)统计分析:系统配备了种类多样的可视化分析工具,其中包括月度报销金 额所呈现的柱状图以及部门报销分布对应的饼图,借助这些工具可协助管理员较为 直观地明晰资金的流向以及报销的模式,除此之外,管理员还有导出详细报销报表 的能力,这些报表可用于开展数据分析工作或者进行存档处理,以此为基于数据的 决策制定提供相应支持。

## 6 研究成果与社会、环境等关系

<span id="page-34-0"></span>近年来,随着人工智能技术的快速发展,图像识别技术在各个领域的应用日益 广泛,尤其在财务管理领域展现出了巨大的潜力。基于图像识别技术的财务报销系 统是我毕业设计中的核心成果,该系统通过自动化的方式实现了财务报销流程的智 能化处理,显著提高了效率和准确性,满足了现代企业对高效、透明财务管理的需 求。

在传统财务报销流程当中,员工需要手动填写报销单据并上传收据,随后由管 理员逐一进行审核,这种方式耗费时间又消耗精力,还容易因人为失误导致错误或 合规性方面的问题出现,而我的系统借助引入图像识别技术,可自动提取收据中的 关键信息,如金额、日期、消费类型等,并且进行合规性检测。这一功能减轻了员 工和管理员的工作负担,提高了报销处理的效率与精确度,为企业节省了大量时间 和人力成本,从社会层面而言,该系统的应用推动了企业数字化转型进程,提升了 财务管理的智能化水平,依靠减少对纸质单据的依赖,降低了企业的运营成本,同 时提高了信息透明度和内部沟通效率。另外系统的智能助手功能为用户提供了便捷 的查询服务,改善了用户体验,促进了企业内部管理的现代化,

在环境方面,该系统的自动化处理大幅减少了纸张和能源的消耗,符合可持续 发展理念,借助 AI 技术对敏感消费行为的检测,系统还可以协助企业遵守相关法规, 避免违规行为,维护企业社会形象。这些特点使该系统是技术创新的成果,是对环 境保护和社会责任的积极贡献,

从技术社会学角度来讲,该系统为企业给予了实际的经济效益,凭借简化报销 流程,企业可节省运营成本,同时借助系统提供的分析功能,管理者能更清晰地了 解财务状况,优化资源配置,这种高效的管理方式提升了企业的市场竞争力。另外 该系统的研发和应用推动了图像识别和自然语言处理技术在财务领域的应用,为相 关技术的发展积累了经验,它的成功为企业管理提供了新思路,也为其他行业提供 了技术参考,呈现了人工智能在实际场景中的广阔潜力。

## 结 论

<span id="page-35-0"></span>本系统将用户认证、报销申请、审核流程、智能助手以及数据分析等多种功能 给予整合,依靠多模态模型与检索提高生成技术的有效融合,实现了票据信息的自 动化提取、动态合规审查以及自然语言交互的协同创新,达成了预期设定的目标, 在测试过程中呈现出良好的稳定性与实用性。

在项目开发过程中运用了多种前沿技术,其中有 Vue.js、Node.js、MongoDB, 以及 OpenAI API 和 Pinecone,这一成果为财务数字化转型给出了创新技术途径,其 模块化设计理念和多技术融合方案对同类系统开发有参考意义,研究提出的技术框 架依靠了行业标准符合性验证,在提升财务管理效能以及推动自动化转型方面呈现 出可行的应用前景。

尽管系统已取得初步成功,不过仍存在一些不足之处,例如用户界面设计可优 化以提升交互体验,系统的安全性与性能也需要持续改进,未来计划增添更多智能 化功能,像自动分类报销项目、预测报销趋势等,以此提升系统的实用性与竞争力。
