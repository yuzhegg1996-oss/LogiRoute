# 摘 要

<span id="page-2-0"></span>随着移动互联网时代的到来,线上购物已成为越来越多大学生的消费方式之一。 而随着互联网、智能终端技术、物联网技术的不断发展,越来越多的大学生将线下 购物转移到了线上购物,这使得大学生的购物行为数据更加丰富多样化。

本课题主要实现一个基于大数据的大学生购物行为分析系统。该系统收集电商 平台各类数据,并进行数据的预处理,可以更好地理解大学生的购物行为,包括购 物偏好、购物心理、消费习惯等方面的信息。本系统采用 Java 开发,前端技术 HTML, 后端使用 Springboot 框架,数据库使用的 MySQL 完成开发。本系统通过协同过滤算 法实现了基于用户行为特征的个性化商品推荐,提高用户的满意度。本系统通过对 购物系统的大学生用户群体的浏览、收藏等行为进行统计分析,实现针对不同用户 个体生成个性化的推荐商品列表,从而可以提供更精准的商品推送服务。

关键词:购物行为分析 大数据 商品推荐 协同过滤

# **Abstract**

<span id="page-3-0"></span>With the advent of the mobile Internet era, online shopping has become one of the consumption methods of more and more college students. With the continuous development of the Internet, intelligent terminal technology and Internet of things technology, more and more college students transfer offline shopping to online shopping, which makes the shopping behavior data of college students more rich and diverse.

This topic mainly realizes the design and application of college students 'shopping behavior analysis system based on big data. The system needs to collect all kinds of data from the e-commerce platform and preprocess the data, which can better understand the shopping behavior of college students, including shopping preferences, shopping psychology, consumption habits and other aspects of information. The system is developed using Java, front-end technology HTML, back-end using Springboot framework and database using MySQL to complete the development. The system implements personalized commodity recommendation based on user behavior characteristics through collaborative filtering algorithm to improve user satisfaction. Through the statistical analysis of browsing, collection and other behaviors of college students in the shopping system, this system can generate personalized recommended commodity lists for different individual users, so as to provide more accurate commodity push services.

**Key words**:Shopping behavior analysis Big data commodity recommendation Collaborative filtering

# 1 绪论

# <span id="page-6-1"></span><span id="page-6-0"></span>1.1 系统背景意义

大学生群体是一个庞大的消费者群体,而且这个群体变化快速,购买习惯和偏 好也在不断发生变化。因此,了解大学生的购物行为对电商平台来说,是非常有意 义的。

通过分析大学生的购物行为,能够更好地把握市场方向,发现产品的销售热点, 提升电商平台的销售转化率和经济效益。同时,也能够了解到学生对某些商品的需 求和偏好,为电商平台提供更加个性化的服务,在购物时更快速准确地找到适合自 己的商品,提高购物体验的同时也减少了时间和精力的浪费,因此对消费者来说具 有一定的便利性和价值。除此之外,对大学生购物行为的分析还能够帮助商家更好 地把握时机,推出折扣活动等促销策略。再次,研究基于大数据的购物行为分析系 统具有一定的学术意义,可以探索大数据和机器学习技术在实际应用场景中的潜力 和可行性,推进科技创新和社会发展,为相关领域的研究和发展提供有价值的启示 和帮助。

总之,基于大数据的大学生购物行为分析系统是一项具有非常重要的背景意义 的研究和实践工作,对大学生购物行为进行深入分析,有利于电商平台深入挖掘市 场潜力,提升服务质量和效率,同时也能够顺应市场变化,迎合大学生群体的消费 习惯和需求。

# <span id="page-6-2"></span>1.2 研究现状

关于电商平台上的再次消费行为的预测问题,吕泽宇、李纪旋、陈如剑等人在 《电商平台用户再购物行为的预测研究》[1]一文中首先针对用户与商家的行为和交易 记录,基于特征工程方法设计了多种不同的行为预测特征,基于可视化等方法对比 分析了预测特征的重要性和特点,进行了属性筛选;然后,基于提出的预测特征设 计使用了多种不同算法训练预测模型。

关于数据分析方面,刘海、宋阳阳、王宝等人在《基于 Hadoop 的购物行为分 析系统的设计与实现》[2]一文中运用 Hadoop 可运行在 Linux 平台上,又由于与 HDFS 完全兼容,可以满足数据处理和分析需求。数据分析完成后,分析产生的数据 将存储在 MySQL 数据库中,以便有序直观地看数据。

杨国胜、郭贝贝在《机器学习增强的电子商务平台用户行为预测》[3]一文中提到 用户行为分析的关键技术,首先需要利用数据挖掘、机器学习、统计学等方法进行 知识的发现,对数据进行特征提取。以此为基础,构建作为知识指导的知识存储与 表示的客户网购行为知识库,进而建立从数据输入到预测行为的系统体系。

刘娅娅、李爽、李晓彤在《基于电商数据的网购感知风险测度及消费者行为分 析》[4]一文中对网购感知风险进行测度,利用谱系聚类法构建消费者网络购物感知风 险指标体系,将网购感知风险划分为五个维度,分别为:来源风险、时间风险、经 济风险、功能风险和社会风险;主成分分析度量每个维度感知风险的大小;贝叶斯 网络模型分析消费者网购行为的产生机制。

在对于大学生网络购物情况调查中,王子松、厉明杰、刘义朔等人在《大学生 网购情况调查及分析》[5]一文中指出网络营销中可能还存在的一些不足,就是学生作 为网络购物消费市场的一大主力,并没有彻底引起各大平台高度的重视,缺乏大学 生市场针对性,产品质量问题严重,个性化产品不突出,难以满足大学生当前消费 心理等问题。

程锐在《基于用户行为特征预测用户的购买意愿和目标商品品类》[6]一文中基于 特征工程理论,对基础数据进行特征提炼,期间通过数据清洗方法清除异常数据, 并通过对基础数据进行必要的数据统计,提炼出四类特征:原始特征、统计特征、 时间间隔特征、计算类特征,特征涉及用户、商品、商品品类、用户评论、用户-商 品对、用户-商品品类范围。完成数据特征的提炼后,通过进一步的特征融合和异常 业务数据处理,生成最终的数据特征。

关于购物行为预测的研究,杨澜在《机器学习在电商用户购买行为预测中的应 用研究》[8]一文中运用数据挖掘技术与机器学习分类算法进行分析,从电商平台健康 发展和效益增速角度出发,研究电商用户的历史交互数据,构建用户对品类下店铺 的购买预测模型,在模型建立的过程中,针对数据的时间特性,利用滑动窗口方法 构造了多类特征,有效提升了模型的预测效能,在基于树模型的几个分类算法预测中, 得出基于 LightGBM 算法的分类模型预测效果最佳。从而帮助电商平台商家更好地 对用户进行个性化推荐、调整商品库存情况等。但仍然存在不足之处,若正负样本

极不均衡,模型的分类性能会很受影响。

王少杰在《基于 Spark 平台的电商推荐系统的研究》[9]一文中研究了融合客服满 意度的协同过滤算法。CS-CF 算法根据 IFK-Means 算法优化基于用户的协同过滤算 法的过程,融入用户的客服满意度,客服满意度会影响用户的兴趣,使用 IFK-Means 对用户的客服满意度进行聚类,得到客服满意度区域,根据欧式距离公式计算用户 与用户之间的客服满意度相似度,并将客服满意度相似度作为权重与用户的皮尔逊 相关系数进行加权和,计算出最终的用户相似度,然后根据用户相似度,预测用户 对其他商品的评分。

薛琳兰在《基于大数据技术的电商推荐系统的设计与实现》[11]一文中提到基于内 容的推荐首先从商品信息表里找到商品标签,并基于内容信息做分词,提取出商品 的特征,然后根据特征给商品做相似推荐,即找到与该商品有相同标签的商品。

李盼颖在《基于协同过滤的个性化推荐算法》[12]一文中提出了一种基于内容和协 同过滤的混合推荐算法,利用基于内容过滤算法的优势,可以过滤掉所有具有相似 性的匹配的项目,特别是当项目没有被任何用户进行评分时,利用协同过滤的优势, 当用户数量和评分等级较大时,在预测协同过滤算法中,用户评分数据的矩阵变得 密集,从而减少了数据的稀疏性。

张岩在《基于 Spark 框架的电商实时推荐系统的设计与实现》[15]一文中提出了基 于 Spark 框架构建的电商实时推荐系统,通过借助 Spark 框架平台及其组件采集用户 行为日志数据,并进行用户隐性行为分析和离线推荐系统训练,提高了离线推荐系 统效率。

#### <span id="page-11-1"></span>2.3.2 基于协同过滤推荐算法分析

协同过滤是一种常用的推荐算法,可以用来对基于大学生购物行为数据的商品 进行推荐。该算法基于消费者的历史行为,寻找与此行为相似的其他消费者,并推 荐这些消费者所在的集合中产生的其他购买行为。

在具体实现时,协同过滤推荐算法可以分为两种方法:

#### (1)基于用户的协同过滤方法

该方法基于用户的行为,评估用户间的相似性,推荐与该用户有过消费行为的 其他用户购买过的商品。例如,用户 A 和 B 都喜欢购买笔记本电脑,那么,当用户 A 购买了一台快速笔记本电脑后,推荐给用户 B 快速笔记本电脑。

#### (2)基于物品的协同过滤方法

该方法基于特定商品被多少用户购买了,评估商品与商品之间的相似性,推荐 与用户喜欢的商品最相似的其他商品。例如,如果用户 A 经常购买一些品牌的女士 衣服,那么会推荐同品牌系列的其他女士衣服。

本系统的商品推荐主要采用皮尔逊相关性系数来计算两个用户之间的相似度。 皮尔逊相关系数:常用于描述两个变量之间线性相关程度的统计量,取值范围为-1 到 +1 之间,接近 0 的变量被成为无相关性,接近 1 或者-1 被称为具有强相关性。

$$p(x,y) = \frac{\sum x_i y_i - n \overline{xy}}{(n-1)s_x s_y} = \frac{n \sum x_i y_i - \sum x_i \sum y_i}{\sqrt{n \sum x_i^2 - (\sum x_i)^2} \sqrt{n \sum y_i^2 - (\sum y_i)^2}}$$

# 3 系统设计

# <span id="page-15-1"></span><span id="page-15-0"></span>3.1 系统功能概述

大学生购物行为分析系统设计与应用的实现是由两部分组成,一个商品用户 端系统,另一个是后台管理系统,两者皆有不同的功能模块组成,进而形成一个 完整的系统,每个功能的内容都是根据需求而设计开发的。

#### (1)登录注册模块

此模块完成了用户可以通过手机号和密码进行注册登录系统的功能。

#### (2)商品推荐模块

此模块完成了商品搜索功能,查看商品详情页功能,个性化商品推荐功能。

#### (3)个人中心模块

此模块完成了个人信息管理,可以修改个人信息,可以添加收货地址,查看个 人历史订单,取消订单功能。

#### (4)购物车模块

此模块完成了对购物车的商品进行管理功能,可以删除购物车里面的商品,可 以编辑商品数量信息,商品结算功能。

#### (5)用户信息管理模块

此模块完成了后台可以实现对用户信息进行管理功能。

#### (6)商品信息管理模块

此模块完成了后台可以实现添加商品功能、商品信息修改功能、商品上架/下架 功能。

#### (7)分类管理模块

此模块完成了对商品的分类进行管理的功能,新增分类、下级分类管理功能。

#### (8)订单管理模块

此模块完成了对商城所有订单进行管理的功能,关闭订单功能、修改订单功能、 配货完成功能、出库功能。

# <span id="page-22-1"></span><span id="page-22-0"></span>4.1 登录注册模块

用户登录:用户可以通过手机号和密码进行登录,以进行商品浏览、购买等操 作。

用户注册:根据传入的用户名和密码创建一个新的用户并保存在数据库中。首 先,通过调用 selectByLoginName() 方法检查管理员提供的用户名是否已经注册,如 果已经注册则返回用户名已存在的错误提示。否则,根据传入的用户名、密码和唯 一生成的 NickName 创建一个新的 MallUser 对 象 registerUser 并调用 MD5Util.MD5Encode() 方法对密码进行加密操作。最后调用.insertSelective() 方法将 新创建的 MallUser 插入到数据库中,如果操作成功则返回成功操作提示,否则返回 数据库错误提示。具体代码实现如图 4-1 所示。

# <span id="page-22-2"></span>4.2 商品推荐模块

皮尔逊相关系数计算公式代码实现:getRelate()方法用于计算给定属性之间的相 关性系数。其中,参数 xs 和 ys 是表示属性值的列表,方法返回 Double 类型的相关 性系数。首先通过 mapToDouble()方法将整数列表转换为 Double 类型,使用 sum() 方法计算列表总和、平方和和乘积之和。然后根据公式计算属性之间的相关性系数, 方法最后返回该系数。具体的代码如图 4-2 所示。

#### 图 4-2 皮尔逊计算公式代码图

计算两个用户之间皮尔逊相似度的方法 pearson\_dis(),采用了双重循环遍历两个 用户的购买列表,然后计算两个用户的购买次数之间的皮尔逊相关性系数。其中, 参数 xList 和 yList 是表示两个用户的商品列表以及购买次数的列表,该方法返回一 个 double 类型的相似度值,方法中还使用了一个叫做 Lists.newArrayList()的工具方 法来创建 Java 集合。具体的代码如图 4-3 所示。

图 4-3 相关系数计算代码图

计算与指定用户最为相似的其他用户的方法 computeNearestNeighbor()的实现, 其中参数 userId 表示指定的用户 ID,参数 list 表示所有用户的商品购买记录。该方 法返回一个 Map 类型的键值对,其中键表示与指定用户之间的相似度,值表示与指 定用户最为相似的用户的 ID。首先将所有用户的购物列表按照用户 ID 进行分组,然 后遍历所有用户的购物列表,计算与指定用户之间的皮尔逊相关系数,然后将其存 储到 TreeMap 对象中。具体的代码如图 4-4 所示。

# 结 论

<span id="page-36-0"></span>本课题设计与实现了一个大学生购物行为分析系统,在商品交易功能的基础上, 通过大数据技术和数据分析算法,根据用户的历史购买记录和浏览行为,为其推荐 更加符合其消费需求的商品。本系统基于 Springboot 框架开发,使用 MySQL 数据库, 主要包括用户管理、商品浏览、商品交易等功能。本系统通过数据分析对购物系统 的大学生用户的特征进行提取,并基于协同过滤算法实现了具有个性化的商品推荐 功能。

当然,该系统还是存在不足之处,虽然基于协同过滤推荐算法简单易用,并且 可以为用户提供社交化的、个性化的推荐服务。但是,协同过滤算法还存在一些缺 点,比如:冷启动问题,该算法需要大量的用户行为数据才能工作,因此,在开始 使用时,可能无法为新用户推荐商品。数据稀疏性问题,对于某些品类的商品,可 能存在很少用户购买这些商品的情况,这就导致无法得到可信的推荐结果,还有简 单的行为分析,协同过滤推荐算法仅仅通过用户的历史行为数据来推荐商品,无法 考虑到用户的其他信息和偏好。本系统实现的功能都是基本功能,要让这个系统可 以运用到实际中还差很多很多,还需要我不断学习、补充和完善,才能更好的服务 用户群体,服务社会。
