# <span id="page-2-0"></span>摘 要

数据挖掘的个性化影视作品推荐系统设计的目的是为会员提供电影信息等方面 的平台。

与其它应用程序相比,个性化影视作品推荐的设计主要面向于电影院,旨在为管 理员和会员提供一个数据挖掘的个性化影视作品推荐系统。会员可以通过系统及时查 看电影信息等。

通过对大量的数据挖掘算法的分析和学习,本系统采用一定的数据挖掘算法和协 同过滤挖掘算法来实现个性化影视作品的基本系统。在本系统中,对相关热门电影和 最新电影进行推荐,从而满足用户无意识基本的核心需求。

协同过滤算法是数据挖掘算法中最应用广泛的算法之一,是本系统的核心算法。 本文采用协同过滤算法中的基于模型的算法和基于全局的算法两种算法。

本系统是基于 Windows 操作系统的,同时为防止出现兼容性及稳定性问题,服务 器选择的是 Tomcat,前台与后台之间的数据存储主要通过 MySQL。用户在使用应用时 产生的数据通过 JAVA 等语言传递给数据库。通过此方式促进数据挖掘的个性化影视 作品推荐系统信息流动和数据传输效率,提供一个内容丰富、功能多样、易于操作的 数据挖掘的个性化影视作品推荐系统。

## <span id="page-7-0"></span>1.2 国内外研究现状

在 1995 年的美国人工智能协会上,卡内基梅隆大学的 Robert Armstrong 团队 提出了个性化导航系统 Web Watcher,同时斯坦福大学的 Marko Balabanovic 团队 也推出了个性化推荐系统 LIRA,麻省理工学院的 Henry Lieberman 提出的个性化导 航智能体 Letizia,这三个经典系统标志着个性化推荐服务的兴起和开端[6]。

虽然在国内,个性化推荐研究滞后于国外,是在 2000 年才开始进行的,但国内 研究者们仍然取得了一定的成果,并且衍生出了许多影视推荐系统。一个影视推荐系 统主要由用户的历史记录、影视推荐算法和决策信息三部分组成,其中最为核心和关 键的是影视推荐算法。目前,影视推荐算法的类型大致可以分为基于协同过滤的影视 推荐算法、基于内容的影视推荐算法、基于关联规则的影视推荐算法、基于知识的影 视推荐算法等四类[7]。但综合当前研究现状发现,每种算法都存在其不足和局限性, 因此影视推荐算法研究面临着巨大的挑战。

协同过滤推荐技术是推荐系统中应用最为广泛且成功的技术之一。该技术通常采 用最近邻算法,通过用户的历史喜好信息计算用户间的相似度,并利用目标用户的最 近邻居用户的加权评价值来预测目标用户对某个特定商品的喜好程度,以此为依据向 目标用户进行推荐。协同过滤技术最大的优点是对推荐对象没有特殊要求,能够处理 如音乐、电影等非结构化、复杂的对象[8]。但是,协同过滤技术也存在着不足,特别 是在用户评分数据较少时推荐质量会有所下降。因此,多种数据和技术的有效整合得 到了研究者们的关注。例如 Balabanovic 等人提出了通过基于内容的过滤与协同过滤 的复合型推荐系统来提高推荐质量。文中还提出了基于 Web 使用挖掘和 Web 内容挖掘 的推荐系统。

<span id="page-8-0"></span>近年来,随着电子商务的快速发展,推荐系统的商业应用范围不断扩大。许多知 名的在线平台如亚马逊、CDNow 和淘宝网等都在运用协同过滤技术来为用户提供个性 化推荐服务。微软研究院开发的协同过滤工具已被集成在微软的 Commerce Server 产品中,并广泛被许多站点使用。因此,协同过滤技术已经成为在线购物网站不可或 缺的一部分,国际上也涌现出了很多著名的个性化推荐系统[9]。

## <span id="page-10-1"></span>2.5 数据挖掘介绍

数据挖掘是指从不完整的、嘈杂的、模糊的和随机的数据中提取以前未知但可能 有用的信息和知识的过程。

用于数据挖掘的数据源必须是真实而庞大的,可能包含不完整和模糊的数据元素。 获得的信息和知识必须对用户来说是有趣和有用的。一般来说,数据挖掘的结果更多 的是识别大趋势,而不是产生完全准确的知识。

<span id="page-10-2"></span>数据挖掘可以简单理解为操纵大量数据以揭示有用知识的过程。数据挖掘是一个 广泛的跨学科领域,包括数学统计、数据库、神经网络、模式识别、机器学习、模糊 集、模糊数学和其他相关技术。

## <span id="page-11-2"></span>3.3 数据挖掘技术分析

本文的影视个性化推荐系统是通过数据挖掘来主动根据用户收藏观看等行为进 行相关电影的推荐的。实际上,是通过用户本身的查看电影的行为和将电影进行收藏 等记录进行数据挖掘的。

通过相关数据挖掘的算法,一般应用协同过滤技术中的最近邻技术,将用户的历 史记录信息通过计算,从而分辨出历史喜好相关的用户之间的距离,从而利用目标对 象的最近邻居对电影评价的加权平均值来预测该对象对其他电影的喜爱深度,系统通 过该面依据这一喜爱深度来对目标用户进行推荐。

# <span id="page-16-2"></span>4.2 系统设计协同算法

协同过滤是一种算法,它使用用户以前的行为、兴趣和偏好的信息,通过计算用 户或对象之间的相似性来进行个性化推荐。基于用户的协同过滤可以识别与当前用户 有相似兴趣的其他用户,并推荐这些用户喜欢的产品;而基于对象的协同过滤则是基 于对象之间的相似性,并推荐与用户相似的产品。

协同过滤被应用于许多领域,如推荐系统、电子商务和社交网络等等,它可以帮 助改善用户体验,增加销售和平台的盈利能力。协同过滤算法使平台能够更好地了解 消费者的需求,并提供更符合他们兴趣和需求的产品,使消费者更容易找到满足他们 的产品,增加零售商的盈利能力。

本系统更偏向于通过用户的收藏行为,对用户看到的页面进行推荐。使用收藏相 关的协同过滤有几个优点:

- (1)收藏作为一种用户行为更准确地反映了用户的兴趣。与传统的基于点击和观 看的行为的协同过滤相比,用户的收藏行为通常能更准确地反映用户的偏好和兴趣, 因为用户通常只收藏他们真正喜欢的电影,而忽略其他他们不感兴趣的电影。因此, 使用收藏行为作为协同过滤算法的基础可以更准确地推荐用户感兴趣的电影。
- (2)可以增加数据丰富性:在影视推荐中,用户对于某个影视作品是否喜欢的评 价可能很少,甚至没有。而用户通过点击收藏行为表示对某个作品的兴趣,为协同过 滤算法提供了更多的数据,增加了数据的丰富性和覆盖面。户展示推荐的依据,使用 户更容易理解推荐结果的产生,提高用户的信任度和用户体验。
- (3)明确向用户展示推荐依据:基于收藏的协同过滤算法可以对用户的历史观影 记录进行统计,明确向用户展示推荐的依据,使得用户更加容易理解推荐结果的产生, 提高了用户的信任度和用户体验。

软件测试指的就是通过自动或是人工的手段来运行某个软件系统,并对其运行过 程进行综合测定,其目的在于对所开发软件系统质量进行合理评判。软件测试的最后 的目的是验证软件的正确性,即是否满足既定的需求分析,是否满足用户的特定功能。 将未满足的需求得以实现或错误改正[15]。根据测试目的的不同,所选取的测试方法也 存在差异。而软件质量保证是为保证软件质量而建立的一套标准,目的是给予管理者 一套管理标准,使软件测试环节系统、有计划的进行,能够及时找到错误。由于软件 本身是无形态的逻辑产品,因此不存在错误是不可能的。工厂的产品在生产出厂前需 要经过严格的测试检测,同样地软件也需要严格的软件测试。它可及时告知开发人员 并修改问题,代表了对需求分析、设计、编码的最终审查。按测试技术不同可划分为: 白盒测试、黑盒测试、灰盒测试[16]。

## <span id="page-34-0"></span>6.1 软件测试原则

软件测试是为了发现错误而执行的程序的过程,它的最终目标是提升软件质量, 提高用户体验。软件测试可能会带来经济效益,因为软件测试可能会发现其中隐含的 错误,若一个小小的错误未及时被发现,那么将来也许会引起更严重错误或者损失。 那么这个损失或者错误将会用更大金钱或者人力来弥补,也可能会引起不可挽回的损 失。俗话说"不以规矩,不能成方圆",一个好的指导原则能指引测试员尽早地发现 软件错误。经过前辈们的不断摸索和实践,总结出以下指导原则供参考:

## (1)应制定测试计划并严格执行,排除任意性。

软件测试覆盖整个软件开发过程,它在软件测试中花费的时间比较多,地位也是 比较重要的。软件测试应该制定严格的测试计划,在测试过程中,应严格按照计划进 行,保证测试稳定进行,不影响最终项目的上线。制定计划时,应考虑到各个影响测 试方面的各个因素,保证其合理性,以及如何针对不同类型的测试团队人员,将测试 执行进度计划做出相应的调整。

## (2)测试用例应由测试数据和预期输出组成。

要测试用例应该包含模拟用户正常输入时的数据,用户理应得到的数据,还有在 测试时用户真实得到的数据。

(3)程序员应避免检查自己的程序。

软件测试其实是一种找出错误的过程,每个人在自测时,有可能会因为个人对需 求文档描述内容理解不当而引入错误更加难发现。避免此行为,有利于保证测试工作 客观性、有效性。

(4)测试用例应当包括合理的和不合理的输入条件。

为了使得避免局限性,软件测试应涵盖到各方面,其中条件就需要包含到合理的 和不合理的。

(5)充分注意测试中的群集现象。

根据数据统计,软件测试出出现的错误有可能中发生群集现象。因为一个模块的 错误越多,其发生错误的可能性越大。因为在改错的过程中,有可能会引入更多的错 误。因此对错误较多的片段要进行下一步更详细的测试。

(6)既要检查是否做了该做的事,还要检查是否做了不该做的事。

进行功能测试时,要注意软件是否准确完成了某个模块。例如,需要打印输出用 户个人信息,而打印出的是电影信息,这显然是不合理的。

(7)妥善保存测试用例。

测试用例的设计耗费了很大时间,必须将其制成文档保存,还应进行出错统计和 最终分析报告。因为经修改后的程序可能还会有新的错误,需要进行不断的回归测试, 测试用例的保存有利于节约时间成本,也为后期的维护提供了便利。

## <span id="page-35-0"></span>6.2 软件测试过程

软件测试这个任务是非常巨大的,有很多的代码需要测试人员完成测试,为了保 证软件测试任务高效、有效率,可以考虑把软件测试过程划分为几个阶段,保证每个 阶段有条不续的进行。主要可以分为以下阶段:

## (1)单元测试

单元测试是对软件的某个模块或者代码中的一个函数进行的测试,是软件测试过 程最先要考虑的测试,即验证某个模块的独立性,看与其他模块想分离出来情况下, 看它是否可以独立地完成它的任务。这个测试过程长期看的话,可以提高开发人员对 代码的重视程度,减少低级错误,而且可以减少以后维护的话费。短时间来看的话, 测试人员的工作量变得很大,对于项目比较紧张或人数较少的公司来说,带来了很大

的负担。

## (2)集成测试

集成测试是对各模块间耦合性的测试。因为在完成模块测试后需要对软件进行组 装起来,然后进行基础测试,难免在组装过程中发生代码的丢失会重复定义等错误, 或模块测试的错误可以被忽略,而集成后,错误变大导致出现很大的错误,所以集成 测试是必须的一个环节。有按照检测顺序将测试分类,例如:自顶向下的集成策略、 回归测试、核心系统先行集成测试;有按照具体特性将测试分类,例如:基于功能、 分险的集成测试等。

## (3)系统测试

系统测试是指将已经集成的软件系统作为整个计算机系统的一个元素,与其他系 统元素结合在一起,在实际运行(使用)环境下,对计算机系统进行一系列的组装测试 和确认测试。这个阶段的目的是与软件需求说明书作比较,检查软件的功能和模块是 否符合说明书的要求。因为在有些时候虽然软件的某个模块的功能虽然满足用户的需 求,也能满足用户的交于的任务,但是组装起来的话,会遇到预料不到的错误。次外, 软件是装在系统上与其他软件一起使用的,在与其他软件使用时是否会发生错误也是 未知的,可见系统测试也是非常有必要的。

## (4)确认测试

任务是检查软件各个模块或者功能是否满足说明书的描述,如果功能和最终的测 试满足用户需求,则说明这个软件还是比较成功的,可以完成交付。相反,则需要用 户和经理进行协商,制定计划如何进行修改的计划。

## (5)验收测试

软件在进行上线或产品发布之前的测试,需要用户和开发人员一起共同合作,来 参与到测试当中。目的是测试软件是否准备就绪,是否可以完成开始某个小功能和最 终的客户需求。

## <span id="page-37-0"></span>6.4 本章小结

软件测试是保证软件质量的最后一个阶段,在项目组花费的精力较多,因为当完成测试后,项目就要正式进入到维护运行阶段。经过测试的软件,发现软件隐含错误时,就需要查找错误原因和位置,即软件调试。软件调试是一次非常困难的获得,目前还为发现十分有效高速的调试方法,因为错误的原因和种类很多。因此,应严格遵守测试原则,积极开展测试工作。

# <span id="page-38-0"></span>结 论

作为一个基于数据挖掘算法的个性化影视作品,数据挖掘的相关技术的重要性是 不言而喻的。数据挖掘的相关算法可以有效的实现个性化影视作品推荐。通过数据挖 掘技术,我们可以从大量的用户的历史行为作为数据并从中提取出用户的偏好和特征, 最后根据这些信息给用户推荐最符合要求的影视作品

同时,本文提出的基于数据挖掘的个性化影视作品推荐具有较高的准确性和实用 性。能够有效的推荐最符合用户偏好的影视作品。项目目前完成了用户基本需求,兼 容性良好,未发生错误。但与真正投入使用的平台相比,功能和实用性明显不足,后 期还有很多改进和完善的地方。此平台的功能丰富和对数据库的设计以及数据的处理 的更加合理高效。各种界面和人性化的设计也值得去认真学习和探索。

在当前的大数据时代,运用数据挖掘技术进行个性化影视作品推荐已经成为一种 必要的趋势。同时,本文提出的基于数据挖掘的个性化影视作品推荐模型也为此提供 了一种有效的解决方案。

系统还有很多可以在未来逐步完善的地方。比如基于用户历史行为这一点,并不 能很准确的反映用户的真实爱好,未来可以通过对用户的社交网络,地理位置,消费 习惯等等进行更综合的推荐。也可以对个性化进行更透明的解释,向用户解释清楚个 性化是基于哪些方面,未来可以借助解释性机器人等进行深度学习培训,从而对推荐 过程进行解释,提高用户的满意度等等。

总而言之,未来将会有更多关于数据挖掘的个性化影视作品推荐系统的研究和实 践,主要是围绕用户的个性需求和体验展开。
