## 摘 要

<span id="page-2-0"></span>人力资源管理系统对于企业来说具有不可或缺的意义,它一方面减轻了人事的 负担,一方面更抽象的展示出企业目前的人力资源的形式,便于企业管理者进行管 理。人力资源管理系统的概念已经出现过多年,目前也近乎完善,但是相较于传统 的人力资源管理系统而言,新员工入职不乏要填写许多的信息,其中一些号码的填 写,例如:身份证、银行卡、各种证件号码等等,由于号码过多,容易填写错误, 所以可以利用图像识别技术进行识别填写。

本次毕业设计将图像识别与人力资源管理系统相结合,利用图像识别的功能辅 助员工进行信息的填写,既节省了填报的时间,提高效率,同时降低了填报的错误 率。系统的实现后端开发主要使用 Go 语言、Go Web 框架使用 Gin 框架、数据库采 用 MySQL 数据库,前端主要使用 HTML、JavaScript、CSS 进行开发。项目开始前 进行可行性分析、需求设计,并对系统的各个角色以及相关职能做一些了解,便于 进行数据库的设计。完成数据库设计后开始进行功能设计。

系统的主要功能需按照人员职能划分为三部分。其中,系统管理员的主要功能 有:部门管理、职级管理、员工管理、权限管理。管理员的主要功能:员工管理、 通知管理、考勤管理、薪酬管理、招聘管理、考试管理。员工在管理员的基础上限 制一些功能,并且新增一个证件识别的功能。

按照软件设计与开发的流程,本论文描述了基于图像识别的人力资源管理系统 需求分析、系统设计与实现、系统功能测试的各个环节。

关键词: 图像识别 人力资源管理 Go Gin

## **Abstract**

<span id="page-3-0"></span>Human resource management system for enterprises has an indispensable significance, on the one hand to reduce the burden of personnel, on the other hand more abstract to show the current form of human resources of enterprises, easy for enterprise managers to manage. The concept of human resource management system has appeared for many years, and it is almost perfect, but compared to the traditional human resource management system, new employees have no shortage of information to fill in, and some of the numbers are filled in, such as: ID cards, bank cards, various document numbers, etc. Because of the excessive number, it is easy to fill in the error, so you can use image recognition technology to identify and fill in.

The graduation project combines image recognition with human resource management system, and uses the function of image recognition to assist employees to fill in information, which not only saves the time of filling in, improves efficiency, but also reduces the error rate of filling in. The implementation of the system mainly uses the Go language for back-end development, the Go Web framework uses the Gin framework, the database uses the MySQL database, and the front-end mainly uses HTML, JavaScript, and CSS for development. Feasibility analysis, requirements design, and understanding of the various roles and related functions of the system before the start of the project facilitate the design of the database. Start functional design after you finish the database design.

The main functions of the system need to be divided into three parts according to the functions of personnel. Among them, the main functions of the system administrator are: department management, rank management, employee management, and permission management. The main functions of the administrator: employee management, notification management, attendance management, compensation management, recruitment management, examination management. Employees restrict some functions on the basis of administrators, and add a new function of document recognition.

According to the software design and development process, this paper describes the requirements analysis, system design and implementation of human resource management system based on image recognition, and system function testing.

## 绪论

## <span id="page-7-0"></span>(1) 国内外发展情况

国内多数公司人力资源管理制度不规范,且价格昂贵,导致了产品实用性、有 效性大打折扣。所以国内一些较大的企业会自行开发人力资源管理软件以便降低成 本,并且可以制作成产品化的代码反复利用,同时在一定程度上能够满足个性化需 求。在完成开发后,基层单位也可以复用。国外的人力资源管理更加强调制度化, 而国内企业受文化影响,更加人性化。这导致国外很难开发出符合国内企业所需要 的产品。

国外在技术培养、与数据库公司合作,并且市场推广上投入较大,强强联手, 使得系统更加完善。混沌管理的特点也体现在现代企业的⼈力资源管理系统中。因 此,根据现代混沌管理的理论和⽅法及其在现代混沌管理中的应⽤经验,可以准确 把握现代企业⼈力资源管理的重点和难点,实现和谐管理和有效管理,从而大大提 高现代企业管理的整体效益。值得强调的是,现代企业⼈力资源管理中的预测与决 策、组织设计和领导力管理是现代混沌管理理论和⽅法的实践领域。同时,该实践 领域更能体现现代混沌管理理论和⽅法在现代企业⼈力资源管理中的具体应⽤。

#### (2) 选题背景和意义

传统人事管理以"事"为中心,只见"事",不见"人"。注重的是一件事有哪些人 参与和事的过程。强调的是以事来"控制人",不关注人和事的整体性。现代人力资 源管理则强调以人为中心,着重于每个人参与某些活动或者人的心理以及意识的变 化。

传统的人事管理被认为是特定职能领域的独立工具,似乎相对独立于其他职能 领域。然而,现代人事管理却截然不同。执行人力资源管理任务的组织的人力资源 部门日益成为决策的重要伙伴,从而加强了它们在决策中的地位。当人力资源管理 涉及到每一位公司经理时,应该明确的是,他们既是该行业的经理,也是人力资源 经理。人力资源部主要负责制定注重能力发展和培训的人事规划和政策,以及培训 其他职能经理或管理人员,以提高人力管理的地位和质量。公司规模越大,人事管 理系统处理的工作量就越多,通过图像识别可以解决数据输入问题。使用图像识别 技术输入信息不仅简单,而且可以降低填写的错误率。

#### (3) 关键理论及技术

## ① Go 语言

Go 语言是由 Robert Griesemer、Rob Pike、Ken Thompson 设计的,是基于 Inferno 操作系统所开发的,是 Google 开发的一种编译型[、静态](https://baike.baidu.com/item/%E9%9D%99%E6%80%81)[强类型、](https://baike.baidu.com/item/%E5%BC%BA%E7%B1%BB%E5%9E%8B)并发型开源[的编](https://baike.baidu.com/item/%E7%BC%96%E7%A8%8B%E8%AF%AD%E8%A8%80) [程语言。](https://baike.baidu.com/item/%E7%BC%96%E7%A8%8B%E8%AF%AD%E8%A8%80)与其他作为学术实验开始的语言不同,Go 代码的设计是务实的。每个功 能和语法决策都旨在让程序员的生活更轻松。Golang 针对并发进行了优化,并且 在规模上运行良好。由于单一的标准代码格式,Golang 通常被认为比其他语言更 具可读性。Go 的自动垃圾收集比 Java 或 Python 更有效。Go 最主要的特征就是 并发,随着硬件的发展,多核处理器的出现,为了提升效率提出的并发,是每个程 序员需掌握的基本技能。Go 推出的 Goroutine,从根本上将一切并发化,运用类协 程的方式处理并发。在运行时用 Goroutine 运行所有的一切。并且使用并发十分简 单,仅一个关键字"go"即可。且无需处理回调,不需要关注线程切换。Go 提倡 使用 channel 通信贡献内存,结合 channel 使用 CSP 模型。

## ② Gin 框架

Gin 框架是基于 Go 的 net/http 库开发的 Web 框架,它提供类似 Martini 的 API, 但性能更佳,速度提升高达 40 倍。较 Go 提供的 net/http 库进行 Web 开发要方便。 Gin 框架以快速著称,基于 Radix 树的路由,小内存占用。没有反射。可预测的 API 性能。Gin 框架支持中间件并且创建中间件十分简单,在路由后使用.Use()方法就可 添加中间件,传入的 HTTP 请求可以由一系列中间件和最终操作来处理。Gin 框架 有 Crash 处理,Gin 捕获异常使用 defer 关键字在函数返回时调用 recover()函数来捕 获 panic。Gin 包含 JSON 验证,提供路由组,可以更好的管理路由,并且这些组可 以无限制的嵌套,不影响性能。中间件也可以直接赋给路由组。

其功能如下:

员工可以查看个人信息和通知,考勤上报、工资上报、招聘信息、考试管理。

#### (4) 登录授权

员工点击编辑按钮,在弹出框内填写需要修改的个人信息,点击弹窗底部的编 辑按钮即可完成对个人信息的编辑操作。

员工可以查看管理员发布的通知,并且可以进行编辑。在通知上方也可以按照 通知标题进行模糊查询。

#### (7) 考勤管理

点击上报按钮在弹窗中填写好考勤信息,点击上报即可上报考勤,已上报的考

勤可以进行编译后可重新上报,也可删除上报的考勤。子菜单可以选择查看考勤历 史和考勤审批。

可以查看工资的明细,和发放的工资的历史。

薪资管理也是如此,管理员可以查看所有员工的薪资水平和具体明细,以及发 放工资。在工资详细中,也可进行添加和删除。

#### (2) 职级管理

点击添加按钮可以对职级进行添加,同样也具备修改和删除功能。

## (3) 员工管理

员工信息管理大致上和管理员相同,员工登陆密码中超级管理员可以通过搜索 员工编号进行定位,通过点击修改密码可以修改员工密码。

角色权限管理可以设置三种角色对应的权限,管理员管理可以对企业员工进行 权限操作,即可将某位普通员工晋升为管理员,同样亦可将管理员权限下放至普通 员工。

## 2 总体设计

## <span id="page-13-1"></span><span id="page-13-0"></span>2.1 系统总体设计

系统主要提供三大模块,一个是普通员工,另一个是管理员,还有一个是超级 管理员。人力资源管理系统主要是便于企业进行人力资源方面的管理,让人事从繁 琐的文件整理中解放出来,从而有更多时间去关注企业员工,为公司的发展做良性 引导。人事长期从事繁杂的文件整理和记录,难免会力不从心导致错误填报和统计, 一方面错误对于员工和企业来说都有影响,另一方面是发现错误时想要处理却不好 查找。人力资源管理系统大大提高了人事的效率,让人事更多以"人"为中心,将 更多的时间花在企业员工身上,而不是以"事"为中心,在琐事中浪费时间。

## <span id="page-13-2"></span>2.2 总体框架设计

本系统采用的是 Gin 框架。同时代码的具体逻辑类似于 Java EE 中的 SSM 框架。 在 Gin 框架中有几个重要的模型。如图 2-1 所示。

Engine: gin.Engine 是一个容器对象,初始化 gin 对象实例,是整个框架基础。 包括路由以及路由组控制、日志、handlercontext、中间件的设置等相关方法。

RouterGroup:定义各种路由规则,gin.RouterGroup 存储所有的中间件。通过 HTTP 将路由注册到 context 上下文实现的 handler 中。

Pool:存储 context 上下文对象,用来优化处理 http 请求时的性能。

Context: 提供 JSON 响应体,实现中间件中变量的共享,也可验证 JSON。一 般在开发中,业务逻辑处理的实现也是在其引用对象中进行。

#### <span id="page-14-0"></span>2.2.1 系统页面视图层

系统前端页面主要使用 HTML、CSS、JavaScript 实现,在前端页面搭建中使用 Layui 框架。Layui 是易上手开源免费的 Web UI 组件库。Layui 组件导入相应 JS 和 CSS 文件后可以直接使用,便于前端页面的搭建,同时也不失美观。

#### <span id="page-14-1"></span>2.2.2 系统应用逻辑层

逻辑层是项目业务处理的核心。系统使用 GORM 框架,所以将 Dao 层部分融 入到 Service 层,包括业务逻辑处理,调用数据层保存数据库,对请求数据参数的 校验。

#### <span id="page-14-2"></span>2.2.3 系统数据层

定义 model 结构体,并在字段后面添加 Tag 标签,便于通过反射查找到对应的 结构体字段,添加 JSON 标签以及 gorm 标签和 binging 标签(表单验证)。不添加 标签,Go 会默认使用反射导出字段名,但字段名需大写才能进行反射,这就导致 导出的字段名与一般的 JSON 样式不符。

## <span id="page-15-0"></span>2.3 数据库设计

#### <span id="page-15-1"></span>2.3.1 数据库系统的选择

本系统采用 MySQL 数据库,首先 MySQL 数据库是开源免费的,并且运行速 度也较快。MySQL 是轻量级关系型数据库,目前属于 Oracle 公司,由 MySQL AB 开发。目前在许多地方都可以见到 MySQL 的身影,因为 MySQL 速度快、开放源 码、体积小且成本低、免费。所以中小型网站在数据库的选择方面都使用 MySQL 作为网站数据库。

MySQL 是关系型数据库,与像 Oracle、SQL Server 等大型数据库相比 MySQL 显得功能有限、规模小。但关联数据库的速度和灵活性较高,数据没有存储在一个 库类,而是将其分散放至各个表中。对于中小型网站来说较于大型数据库的不足之 处都可以忽略,因为开源免费对于中小型网站来说更受欢迎。

## 5 研究成果与社会和经济的关系

<span id="page-42-0"></span>随着企业规模的增大,人力资源管理系统显得更为必要。传统的人力资源管理 系统效率不高,导致人事更多的时间在整理资料,更多的去选择记录企业发生了什 么事,再记录将事中的人从而将二者联系起来。这样的以事为中心以及不可取,现 代的人力资源管理系统更多以人为中心,围绕着每个人来建立,记录每个人发生或 参与了什么事,更加直观的看到每个人所参与的活动。人力资源管理系统的出现, 也大大提高了人事的效率,自动化的整理记录使他们有更多时间关注企业员工,并 且管理人员也可以通过系统看到企业的发展,利于企业指定发展方向。传统的人事 管理被认为是特定职能领域的独立工具,似乎相对独立于其他职能领域。然而,现 代人事管理却截然不同。执行人力资源管理任务的组织的人力资源部门日益成为决 策的重要伙伴,从而加强了它们在决策中的地位。

公司规模越大,人事管理系统处理的工作量就越多,通过图像识别可以解决数 据输入问题。使用图像识别技术输入信息不仅简单,而且可以降低填写的错误率。 让数据录入更加轻松。本系统包括了人力资源管理几个大概的方面,员工管理便于 人事去管理企业员工的个人信息,考勤管理则可以清晰看出员工的出勤状况。招聘 管理列举出企业的岗位,并且还可以对候选人进行管理。在数据存储方面,员工工 号都是随机生成的,作为其登陆账号,初始密码与账号相同。密码可以修改,并且 以 32 位 MD5 加密进行存储,保证了安全性。融入图像识别技术大大提高了信息的 录入率。

## 结 论

<span id="page-43-0"></span>本系统基于 GO、Gin、GORM、HTML、CSS、JavaScript 等技术,使用 GoLand 编 译器开发,完成了人力资源管理系统的设计。从需求可行性、经济可行性、技术可 行性等方面进行分析后,提出了本系统的功能需求。

本系统主要面向企业管理,为企业提供管理企业员工信息的功能和管理企业自 身信息的功能。提供了普通员工、管理员、超级管理员三种角色,三种角色有不同 的需求和权限。基本满足各个角色的需要,并且整体风格简约,操作简单易上手。 系统有以下优点:

- (1) 操作简单易上手,整体界面风格一致且简约。
- (2) 网页相应速度快,运行流畅。
- (3) 易于升级和维护,维护容易。

系统并非十分完美,在图像识别这项技术的融入方面还不够完善。系统并没有 调用外部 API 接口,这也就导致自身的图像识别并不充足和具有说服力。相较之下, 图像识别的准确率不高。真正要做到的图像识别与人力资源管理系统的融合还需与 硬件结合使用。这样可以实现考勤的人脸识别以及证件照扫描登陆识别,以及一些 高精度的图像识别。
